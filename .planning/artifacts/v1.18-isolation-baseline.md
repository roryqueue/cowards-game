# v1.18 Isolation Baseline and Threat Model

**Milestone:** v1.18 Runtime Isolation and Multi-Language Exhibition Beta  
**Baseline inherited from:** v1.17

The v1.18 floor is unchanged from v1.17: normal topology is `web frontend -> Go backend -> Strategy Execution Service / Runtime Broker -> isolated runtime implementation(s)`. JS/TS support remains intact through the registered broker/runtime ABI. Python remains a runtime implementation only.

## Threat Model

v1.18 treats Strategy code as hostile in the non-counted exhibition beta context. The in-scope threats are filesystem escape, network escape, package/import escape, shell/process escape, environment leak, memory/output flood, timeout, crash or malformed IPC, stderr/stack/path leaks, and public-output privacy leaks.

Current subprocess behavior is readiness evidence, not production sandbox certification. Python isolated-mode launch, empty environment, no shell, byte caps, and deterministic failure taxonomy improve the boundary, but they do not prove kernel, namespace, or production hostile-code isolation.

## Promotion Gates

Python may promote only to **non-counted exhibition beta** after the signed-in proof and monitors pass. Python is not ranked, not counted, not broad production multi-language support, and not eligible for ladder or gauntlet counted evidence.

Runtime isolation remains **not production sandbox certification** in v1.18 unless evidence genuinely exceeds the milestone target. Production sandbox certification, arbitrary package installs, PyPI support, WASM/WASI promotion, and cloud/Kubernetes/service-mesh work are out of scope.

No silent fallback is allowed; the gate is **no silent fallback**. A stopped runtime service or stopped Python runtime must fail visibly and must not route Strategy execution into JS/TS fallback, web/API, or Go.

## Ownership

Go owns orchestration, persistence-facing backend behavior, Match lifecycle, scoring, and public evidence. The Strategy Execution Service / Runtime Broker owns schema-validated hostile Strategy execution dispatch. Python owns only non-counted exhibition beta Strategy method execution behind the broker/runtime ABI.
