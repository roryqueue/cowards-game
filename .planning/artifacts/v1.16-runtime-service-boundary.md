# v1.16 Runtime Service Boundary

## Strategy Execution Service / Runtime Broker

The future-facing public abstraction is the **Strategy Execution Service / Runtime Broker**. It is a broker-ready contract for hostile Strategy execution using schema-validated JSON envelopes and the shared runtime ABI.

The v1.16 implementation remains the **isolated JS/TS runtime service** in `apps/runtime-service`. It is not a backend and it is not the final Runtime Broker. It is the current HTTP+JSON binding for `POST /execute-match` and `GET /health`.

## Contract Versions

| Surface | Version |
| --- | --- |
| Runtime execution service | `runtime-execution-service-v1.15` |
| Strategy runtime ABI | `strategy-runtime-abi-v1.14` |
| Request schema | `RuntimeExecutionServiceRequestSchema` |
| Response schema | `RuntimeExecutionServiceResponseSchema` |
| Current transport binding | HTTP+JSON |

The boundary is transport-neutral even though today it runs over HTTP+JSON. A future broker may front or replace this binding without changing Go orchestration, persistence, scoring, or public evidence ownership.

## Authority

The runtime service may validate runtime service requests, execute JS/TS Strategy code through the runtime ABI, and return either internal runtime results or redacted schema-valid system failures to Go.

The runtime service must not claim jobs, complete Matches, persist Chronicles, refresh MatchSet scoring, serve product API routes, read web session state, deliver public evidence, or fall back to retired TypeScript backend paths.

Go remains the owner of normal orchestration, Chronicle persistence handoff, MatchSet scoring/status refresh, and public replay/evidence projection after runtime execution returns.

## Strategy Revision Artifact Policy

Strategy Revisions should be validated, compiled or transpiled where practical, hashed, size-checked, and packaged before immutable Match execution. Matches execute immutable Strategy Revision artifacts or revisions, not mutable source evaluated in Go or web/API processes.

Go and web/API processes must not import, evaluate, transpile, or execute Strategy source. Counted language runtimes must use the same JSON/runtime ABI and schema-validated request and response envelopes; language-specific shortcuts are not allowed.

## Failure And Privacy

Malformed input, ABI drift, timeout, invalid output, execution crash, oversized payloads, source hash mismatch, source byte mismatch, and unsafe diagnostics fail closed.

Failure responses must be schema-valid and redacted. They must not expose Strategy source, StrategyMemory, SoldierMemory, objective payloads, owner debug, raw Awareness Grid, stack traces, stderr, sessions, tokens, DB DSNs, host paths, or private runtime internals. Successful execution results are marked `internal_runtime_result` and are not public replay projections.

## Readiness And Non-Promotion

Worker-thread, subprocess, container-subprocess, non-JS, WASM/WASI, and component-model statuses remain explicit readiness or evaluation labels. No deferred idea is promoted in v1.16.

Node `node:wasi` is not accepted as an untrusted Strategy sandbox. WASM/WASI and the component model remain future evaluation paths, not a production sandbox replacement and not counted non-JS play.
