# v1.16 No-TypeScript-Backend Topology

## Normal Topology

```text
web frontend -> Go backend -> isolated JS/TS Strategy runtime service
```

Allowed TypeScript production-ish roles are limited to the web frontend and the isolated JS/TS Strategy runtime service.

## Strict Command

```bash
pnpm topology:check -- --require-v1-16-no-typescript-backend --json
```

This mode requires web health, representative page-load smoke, Go health, runtime-service health, selected Go page smoke, web-through-Go public Strategy reads, v1.15 lifecycle evidence, and v1.16 no-TypeScript-backend artifact checks.

## Boundary Monitor Command

```bash
COWARDS_REQUIRE_LIVE_TOPOLOGY=1 pnpm boundary:monitors
```

Default boundary monitors keep live topology optional for routine local checks, but the live-required mode uses the v1.16 no-TypeScript-backend topology gate.

## Failure Drills

- Stopped Go must fail closed with no TypeScript service/backend fallback.
- Stopped runtime service must classify the failure as a system failure with no Strategy execution in Go, web/API, or a retired TypeScript backend.
