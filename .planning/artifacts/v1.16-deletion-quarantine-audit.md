**Milestone:** v1.16 Runtime Isolation and TypeScript Backend Retirement

## Selected Normal Role Check

The final label artifact contains 61 `selectedNormal=true` rows. Every selected-normal row is limited to `frontend-only`, `runtime-service`, or `runtime-adapter`, and every row has `normalBackendAuthority=false`.

## Accepted Deferred Surfaces

- Workshop validation, private source, test launch, analytics rerun, profile save, export, runtime support, and UI support.
- Broader ladder mutation/scheduling surfaces.
- Governance/admin mutation surfaces.
- Owner-debug replay projection and owner-private replay paths.
- Migration/schema ownership.
- Production hostile-code sandbox promotion.
- True language-neutral Runtime Broker implementation.
- Counted non-JS play.

## Removed or Retired From Normal Runtime

- Selected account/session, Strategy Revision, fork, exhibition, public read, replay metadata, replay evidence, Match lifecycle, Chronicle persistence handoff, scoring completion, and public evidence behavior are Go-owned or fail closed.
- TypeScript worker lifecycle code is quarantined behind explicit rollback/test/parity purposes.
- `@cowards/service` is parity, fixture, rollback reference, or deferred support only.
- Next.js selected normal routes are frontend adapters to Go, not normal TypeScript backend owners.

## Privacy and Fallback Status

- Public-output denylist remains enforced for Strategy source, StrategyMemory, SoldierMemory, objective payloads, owner-debug data, raw Awareness Grid, stack/stderr, sessions/tokens, DB DSNs, host paths, and private runtime internals.
- Owner-debug replay cannot serve as public replay evidence fallback.
- Test-support and fixture routes are unavailable in normal product runtime.
- Runtime service has no DB/job/public API authority and communicates through the runtime ABI.
