# v1.13 Cutover Scope Decision

**Captured:** 2026-05-23
**Milestone:** v1.13 Go Backend Ownership Cutover

## Decision

Use the aggressive cutover scope.

Go should become the primary backend owner for the product API surface needed by normal workflows, including selected public reads, owner/session reads, auth/session mutations, account Strategy Revision writes/forks/source retrieval, and exhibition MatchSet creation.

## Rationale

There is no live usage, so runtime cutover risk is acceptable. The limiting risks are not user migration or rollback complexity; they are privacy, schema drift, deterministic boundaries, hostile Strategy isolation, and public-output safety.

v1.12 already proved route-scoped switching, fail-closed behavior, raw privacy scans, topology evidence, and rollback for one public read. v1.13 should use that foundation to move real ownership, beginning with Go DB access and live DTO assembly before attempting mutations.

## Hard Boundaries

- Do not execute Strategy code in Go or the web/API process.
- Do not use Node `vm` as a security boundary.
- Keep worker job claiming/completion, Match execution, Chronicle generation, and runtime failure handling TypeScript-owned unless a later milestone explicitly promotes them.
- Keep public/service/Go/topology/monitor outputs free of Strategy source, StrategyMemory, SoldierMemory, objective payloads, owner debug, raw Awareness Grid, stack traces, stderr, sessions, tokens, host paths, DB DSNs, and private runtime internals by default.
- Owner-private source retrieval may return Strategy source only through authenticated private routes and must never enter public or evidence artifacts by default.

## Selected Primary Route Families

- Public Strategy page
- Public player page
- Public ladder page
- Public MatchSet summary
- Public replay metadata
- Auth session read
- Session create/revoke
- Account Strategy Revision list
- Account Strategy Revision source
- Account Strategy Revision save/create
- Starter and Advanced Strategy fork
- Exhibition MatchSet creation

## Deferred By Default

- Match orchestration/job claiming/completion
- Strategy execution and runtime migration
- Full replay projection and owner-debug/private Chronicle assembly
- Go-owned migrations/schema ownership
- Counted non-JS play
- Production sandbox promotion
