# v1.17 Runtime Broker Registry

The v1.17 Runtime Broker is implemented inside the existing isolated Strategy Execution Service. It does not introduce a separate broker service and does not change the normal v1.16 topology:

`web frontend -> Go backend -> isolated runtime service(s)`

## Selection Policy

Runtime selection is exact-match and fail-closed:

- Strategy runtime ABI must be `strategy-runtime-abi-v1.14`.
- Runtime execution service contract must be `runtime-execution-service-v1.15`.
- Language id/version, runtime target, adapter id/version, and package policy must match a registered entry.
- Unknown, stale, or unavailable runtime metadata fails with no JS/TS fallback, no Go execution, and no TypeScript backend fallback.

## Registered v1.17 Entries

| Language | Runtime target | Adapter | Readiness | Counted |
| --- | --- | --- | --- | --- |
| JavaScript | runtime-js | runtime-js-worker-thread | local-dev-fallback | yes |
| TypeScript | runtime-js | runtime-js-worker-thread | local-dev-fallback | yes |
| Python 3.9 | runtime-python | runtime-python-subprocess-experimental | experimental | no |

## Authority Boundary

Go remains the owner of Match orchestration, persistence handoff, completion, scoring, and public evidence. The runtime service owns only schema validation, registered runtime selection, Strategy method execution through ABI envelopes, and internal result/failure return. Python has no route, persistence, job lifecycle, scoring, or public replay/evidence authority.
