# v1.16 Promotion Decision

**Decision:** `promote-no-typescript-backend-except-frontend-and-isolated-js-ts-runtime-service`

## Meaning

v1.16 promotes the normal product topology:

```text
web frontend -> Go backend -> isolated JS/TS Strategy runtime service
```

TypeScript is no longer accepted as the normal backend for orchestration, persistence-facing API behavior, Match lifecycle, Chronicle persistence handoff, scoring completion, public evidence delivery, or silent fallback.

## Accepted TypeScript Roles

- Frontend and frontend Go adapters.
- Isolated JS/TS Strategy runtime service invoked through the public runtime ABI.
- Runtime adapter support code.
- Test, fixture, parity, rollback, quarantined lifecycle, and explicitly deferred support paths documented in the final surface labels.

## Gate Evidence

- Strict v1.16 topology passed with `ok=true` and 27 required checks.
- `pnpm boundary:monitors` passed.
- Live-required boundary monitor passed with 27 required live v1.16 no-TypeScript-backend diagnostics.
- Phase 107 final surface label monitor covers 185/185 inventory rows and rejects semantic/privacy drift.
- Replay visual realism passed with the explicit test fixture gate enabled.

## Accepted Deferred Scope

The milestone intentionally does not remove or replace JS/TS Strategy execution support. It also defers Workshop migration, broader ladder/governance migration, full owner-debug replay migration, database migration/schema ownership, production sandbox promotion, true Runtime Broker replacement, and counted non-JS play.

## Non-Promotion Notes

WASM/WASI/component-model remain long-term candidates for some languages, not v1.16 promotion criteria. Node `node:wasi` is not treated as a sandbox for untrusted code. Container subprocess runtime evidence remains evidence-only until future sandbox promotion criteria pass.
