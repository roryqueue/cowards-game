# v1.24 Runtime Abuse Lab Harness Design

This note is read-only implementation guidance for the v1.24 owner. It proposes a concrete evidence harness shape using existing runtime scripts and exported functions. It does not change source code.

## Recommended New Script

- `scripts/evaluate-v1-24-runtime-abuse-lab.ts`
- Package scripts:
  - `runtime-abuse:evaluate`: `pnpm exec tsx scripts/evaluate-v1-24-runtime-abuse-lab.ts`
  - `runtime-abuse:evaluate:check`: `pnpm exec tsx scripts/evaluate-v1-24-runtime-abuse-lab.ts --check`

The script should be mostly an aggregator plus thin language adapters. Reuse current probe runners instead of duplicating runtime mechanics.

## Core Evidence Schema

Use stable sorted JSON like current evaluators.

Top level:

- `schemaVersion`: `v1.24-runtime-abuse-evidence`
- `milestone`: `v1.24`
- `generatedAt`
- `baselineMilestone`: `v1.23`
- `activeExecutionAbi`: `wasi-preview1-stdin-stdout-json`
- `topology`: `web frontend -> Go backend -> Strategy Execution Service / Runtime Broker -> isolated runtime implementation(s)`
- `claims`: `{ productionSandboxCertified: false, jsTsCounted: true, pythonStatus, rustStatus, zigStatus, directExportsPromoted: false, componentModelWitPromoted: false }`
- `summary`: counts by `status`, `runtimeLane`, `failureClass`, `privacyClass`, and `promotionImpact`
- `lanes`: one object each for `js-ts`, `python`, `rust-wasm-wasi`, `zig-wasm-wasi`, `direct-export`, `component-model-wit`
- `probes`: cross-lane flat list
- `noFallbackDrills`
- `privacy`: `{ publicSafe: true, omittedCategories, checkedWith: "assertPublicOutputLeakSafe" }`

Probe fields:

- `id`
- `runtimeLane`
- `languageId`
- `adapterId`
- `probeClass`: `capability | resource | abi | artifact | unavailable | malformed_output | failure_class | privacy | no_fallback | readiness | signed_in`
- `abuseCategory`: e.g. `filesystem`, `network`, `process_shell`, `imports_packages`, `environment`, `time_random`, `timeout`, `memory_pressure`, `stdio_cap`, `result_cap`, `malformed_json`, `invalid_schema_action`, `crash_trap_abort`, `stale_artifact`, `missing_artifact`, `mismatched_target`, `mismatched_runtime`, `mismatched_abi`, `unavailable_runtime`, `private_data_redaction`

- `observedOutcome`: `{ kind, code, publicMessage, observed }`
- `failureClass`: `strategy_failure | system_failure | unsupported_unavailable | preflight_failure | proof_spike_non_promotion`
- `privacyClass`: `public_safe | internal_redacted | private_not_serialized`
- `promotionImpact`: `none | preserves_js_ts_counted | preserves_non_counted_beta | blocks_promotion | future_work_only`
- `noFallbackObserved`
- `publicSafe`
- `sourcePrivate`
- `reusedFunction`
- `evidenceKind`: `live | preflight | synthetic | signed_in | documentation`

## Direct Export and WIT Guidance

Do not route Match execution through these spikes. They should create fail-loud non-promotion artifacts when tooling or parity is incomplete.

Direct-export artifact fields:

- `abiCandidate`: `direct-export`
- `status`: `proof-only-not-promoted | unavailable`
- `rustParity`, `zigParity`
- `memoryOwnership`: input buffer owner, output buffer owner, allocator owner, free responsibility
- `encoding`: JSON UTF-8 or other candidate
- `caps`: timeout/fuel, memory, input bytes, output bytes
- `validation`: schema and action validation plan/result
- `failureBehavior`: malformed output, invalid schema, trap, timeout, cap exceeded
- `compatibilityWithPreview1Json`
- `matchExecutionEligible: false`

WIT artifact fields:

- `abiCandidate`: `component-model-wit`
- `status`: `proof-only-not-promoted | unavailable`

## Monitor Integration

Extend `scripts/check-boundary-monitors.ts` only after the source owner takes over. Recommended checks:

- `v1.24-runtime-abuse-evidence.json` has all required lane ids and no failed required probes.
- Every probe has `publicSafe: true` and passes `assertPublicOutputLeakSafe`.
- Unsupported/unavailable probes use `failureClass: unsupported_unavailable` and never count as sandbox passes.
- Readiness matrix has direct exports and WIT as not promoted.
- ABI decision keeps `wasi-preview1-stdin-stdout-json` active unless an explicit future promotion exists.
- Signed-in proof includes JS/TS counted, Python beta, Rust beta, Zig beta, result/replay pages, plausible replay start, and private marker scans.
