## Findings Fixed

| Severity | Finding | Fix |
| --- | --- | --- |
| Blocker | Complete fixtures advertised replay availability even when no replay evidence existed. | Fixture summary creation now defaults replay availability to false; only the public-safe replay fixture sets it true. |
| Blocker | Blocked/stale Match rows could be projected as generic no-result failures. | Match failure categorization now maps blocked rows to `blocked` and stale aggregate evidence to `stale_artifact`. |
| Blocker | Aggregate lifecycle override leaked into every per-Match projection. | Per-Match projection now derives lifecycle from each Match row instead of aggregate metadata overrides. |
| Warning | Fixture adapter could be accidentally enabled in production with an env flag. | Fixture enablement now fails closed when `NODE_ENV=production`. |
| Warning | Replay DTO contract was not carried through the replay page server boundary. | `ReplayReadyDto` now carries `MatchExecutionReplayEvidenceV1` and replay evidence copy renders contract lifecycle rows. |
| Warning | Browser proof did not initially cover the full fixture catalog. | Playwright proof now visits every v1.25 MatchSet fixture and the public-safe replay page. |

## Residual Intentional Instability

- Legacy persisted Chronicle replay loading still exists for local/live Matches outside the v1.25 fixture/Go public read path.
- Runtime-service internal envelopes, raw diagnostics, and implementation details remain execution-internal and are not frozen.
- Owner/test-only debug payloads are not part of the app-facing public contract.
