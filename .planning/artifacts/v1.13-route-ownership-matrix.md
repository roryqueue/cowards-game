## Non-Negotiable Ownership Rules

- TypeScript service behavior is the parity oracle and explicit rollback reference, not silent fallback in Go-selected evidence paths.
- Strategy execution, job claiming/completion, Match execution, Chronicle generation, scoring completion, runtime failure classification, and Go-owned migrations are not promoted by v1.13.
- Starter/Advanced fork routes are not selected by default in `COWARDS_GO_ACCOUNT_REVISIONS`; they remain TypeScript-owned until Go can read the library source manifests with parity.
- Public and evidence outputs must omit Strategy source, StrategyMemory, SoldierMemory, objective payloads, owner debug, raw Awareness Grid, stack traces, stderr, sessions, tokens, host paths, DB DSNs, and private runtime internals by default.
