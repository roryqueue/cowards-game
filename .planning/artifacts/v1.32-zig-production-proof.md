# v1.32 Zig Production Proof

## Decision

Zig is promoted to counted eligible through the Zig WASM/WASI language provider path.

## Provider Boundary

- Provider id: `strategy-language-provider-zig-wasi`
- Contract: `strategy-language-provider-contract-v1.32`
- Runtime adapter: `runtime-wasm-wasi-wasmtime-preview1`
- ABI: WASI Preview 1 stdin/stdout JSON, `strategy-runtime-abi-v1.14`
- Target triple: `wasm32-wasi`
- Execution owner: runtime-service

## Counted Eligibility Requirements

Counted Zig entry requires:

- Exact Zig runtime metadata.
- Runtime-service provider validation.
- HMAC proof bound to provider id, contract version, source hash/bytes, artifact hash/bytes.
- Immutable compiled artifact metadata.
- Decoded artifact bytes that match the stored artifact hash and byte count.
- WASI Preview 1 stdin/stdout JSON ABI metadata.
- `validationStatus: valid`.

## Non-Claims

This proof does not claim broad sandbox certification for arbitrary WASM/WASI programs, broad Zig std/package support, or safe execution outside runtime-service / Runtime Broker / provider boundaries.

Focused spec, runtime-wasm-wasi, runtime-service, persistence, web, and Go verification passed for Phase 227.
