# v1.31 Route and Link Inventory

**Milestone:** v1.31 Public Site Spine and Discovery Reads
**Phase:** 211 Route and Link Inventory
**Date:** 2026-05-31

## Boundary

This inventory is app/public UX and public discovery-read work only.

- `match-execution-app-v1` remains frozen.
- Existing public execution DTOs are not changed.
- New discovery reads must be separate public/account-safe APIs.
- No Go match execution, runtime-service behavior, retry/recovery policy, quarantine semantics, job lifecycle, MatchSet scoring, Chronicle persistence, or internal operator controls are changed.
- Public discovery output must not expose Strategy source, StrategyMemory, SoldierMemory, objective payloads, raw diagnostics, host paths, env values, tokens, DB details, package paths, private runtime internals, quarantine details, operator action details, or recovery payloads.

## Existing Public Read Boundaries

### Web public boundary

`apps/web/lib/public-service-boundary.ts` currently exposes:

- `getPublicMatchSetResult(matchSetId)`
- `getPublicReplayMetadata(matchId)`
- `getPublicStrategyCard(strategyId)`
- `getPublicPlayerProfile(handle)`
- `getPublicLadderSeason(seasonId)`

These are individual object reads. They do not provide public home, watch, competition index, competition detail aggregation, or signed-in entry dashboard data.

### Public route ownership

`apps/web/lib/public-service-adapter.ts` has selected public read route ids:

- `getPublicStrategyPage`
- `getPublicPlayerPage`
- `getPublicLadderSeason`
- `getPublicMatchSetSummary`
- `getPublicReplayEvidence`
- `getPublicReplayMetadata`

This is useful precedent for selected Go-backed public reads, but v1.31 discovery reads should not be slipped into `match-execution-app-v1` or existing execution DTOs.

### Public service DTOs

`packages/spec/src/service.ts` has individual page/service DTOs:

- `PublicMatchSetSummaryServiceDto`
- `PublicReplayMetadataServiceDto`
- `PublicReplayEvidenceServiceDto`
- `PublicPlayerPageServiceDto`
- `PublicLadderPageServiceDto`
- `PublicStrategyPageServiceDto`

`packages/spec/src/competition.ts` has public page payload DTOs:

These can be referenced or summarized by new discovery DTOs, but existing fields should not be expanded for discovery.

## Navigation Gaps

- Public visitors currently land in Workshop rather than a public competitive home.
- There is no global site shell in `apps/web/app/layout.tsx`.
- Object pages use local action links, but there is no consistent Home/Watch/Competitions/Learn/Workshop/Account navigation.
- Account page links to public profile, public Strategy cards, source, and exhibition creation, but not competition opportunities or replay/result history as a coherent journey.
- MatchSet and replay pages link to each other, but not back to Watch, competition detail, Learn/trust explanations, or player/Strategy context consistently.
- Player and Strategy pages expose some evidence links, but they do not place those links in a broader competition/discovery context.
- `/ladder/[seasonId]` exists as a competition-like detail page, while desired v1.31 routes use `/competitions/[competitionId]`; Phase 216 should decide canonical mapping/redirect/linking without breaking existing ladder URLs.
- `/exhibitions/new` exists as a creation route, while desired v1.31 entry route is `/competitions/[competitionId]/enter`; Phase 217 should preserve existing creation behavior while giving signed-in users a clearer entry route.

## Discovery Read Candidates

### `getPublicHomeDiscovery`

Purpose: public front door aggregate.

Likely sections:

- Featured/recent public MatchSets.
- Replay-ready Matches.
- Active competitions and entry opportunities.
- Notable player/Strategy links.
- Learn/trust links.
- Empty/unavailable state metadata.

### `getPublicWatchIndex`

Purpose: latest public evidence feed.

Likely sections:

- Recent MatchSets with result status and replay coverage.
- Recent Matches with replay availability.
- Queued/running/degraded/failed/stale/missing/no-result public states.
- Canonical links to MatchSet, replay, player, Strategy, and competition detail pages.

### `getPublicCompetitionIndex`

Purpose: public competition directory.

Likely sections:

- Active ladders.
- Exhibitions/open entry opportunities.
- Tournament placeholders only where backed by real data.
- Public status labels and entry hrefs.

### `getPublicCompetitionDetail`

Purpose: canonical competition detail.

Likely sections:

- Competition identity/status/rules.
- Entrants.
- Standings.
- Schedule/pods/bracket where applicable.
- MatchSets and replay coverage.
- Entry href/state.

### `getSignedInCompetitionEntryDashboard`

Purpose: signed-in account-safe entry flow.

Likely sections:

- Current user/session state.
- Eligible saved revisions with source-free metadata.
- Invalid/stale/already-entered/ineligible states.
- Entry action metadata and links to Workshop/Account.
