# v1.12 Promotion Decision

**Captured:** 2026-05-23

## Decision

`promote-none-yet`

## Rationale

The project now has a route-scoped switch and typed Go client for
`getPublicStrategyPage`. The switch defaults to TypeScript, fails closed when Go
is selected, validates through canonical schemas, scans raw JSON before schema
parsing, exposes sanitized failure classes, and has rollback by removing one
route-specific config flag.

The route is not promoted because the Go backend still serves the public
Strategy page from committed parity fixtures. v1.12 requires live or
production-equivalent data ownership before production web reads can be claimed
as Go-owned.

## Deferred Work

| Category | Deferred item |
| --- | --- |
| Go data provider | Add a read-only production-equivalent provider for the selected public Strategy route |
| Additional reads | Repeat route ownership, parity, topology, no-fallback, and rollback criteria per route |
| Mutations | Keep writes, sessions, ladder mutation, jobs, migrations, and persistence ownership in TypeScript |
| Runtime | Keep runtime isolation promotion and non-JS counted play evidence-only |
| Source-bearing flows | Keep private source retrieval and execution outside Go promotion |
