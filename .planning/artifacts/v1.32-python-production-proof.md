# v1.32 Python Production Proof

## Promotion Decision

Python is promoted to counted eligible Strategy support through the constrained `strategy-language-provider-python` provider path.

## Evidence

- Runtime-service `/validate-strategy` owns Python validation for account saves and rejects imports, dynamic execution, host capability markers, missing required methods, syntax errors, oversized source, and unsafe diagnostics.
- Runtime-service signs provider validation with an HMAC proof over provider id, contract version, source hash, and source byte count; counted-entry gates verify that proof before allowing Python into counted play and fail closed without configured provider secret.
- Workshop Python submissions also require runtime-service `/validate-strategy`; submitted saves cannot locally mint provider provenance.
- Local Python/Workshop builders do not mint `providerValidation`; runtime-service is the provenance authority.
- Workshop persistence recomputes source hash and byte count from submitted source before accepting provider validation.
- Runtime execution remains behind runtime-service / Runtime Broker / Python provider boundaries.
- Runtime metadata requires ABI `strategy-runtime-abi-v1.14`, language `python@3.9`, adapter `runtime-python-subprocess-experimental@0.1.0-experimental`, package mode `none`, no required capabilities, no filesystem, and no network.
- Counted entry gates accept Python only through exact runtime metadata plus `providerValidation` provenance for the same source hash and source byte count.
- Bundled Python starter samples execute through the provider runtime, including helper-function samples.
- Public result/evidence surfaces count Python as counted and keep private Strategy/runtime data out of public output.

## Claims Not Made

- No Python package ecosystem is supported.
- No broad sandbox certification for arbitrary Python programs is claimed.
- No web/API/Go Strategy execution is introduced.
- Rust and Zig are not promoted by this artifact.
