# v1.21 Code Review Remediation

Two review agents audited runtime/spec and Go/Web/Workshop integration. Findings were treated as blockers unless explicitly residual.

## Fixed Blockers

- Artifact-backed WASM/WASI runtime envelopes no longer send Strategy source text to hostile WASM modules.
- Runtime execution now rejects invalid artifact metadata, stale hashes, wrong target/profile/envelope, and source/artifact mismatches.
- Compiled WASM imports are parsed and denied unless they match the narrow WASI Preview 1 stdin/stdout/process-exit plus empty-environment allowlist.
- Rust compile/validation moved out of web/API and Go. Rust submission uses runtime-service `/validate-strategy`; Go and web persist returned metadata only.
- Unknown `sourceFormat` values now fail closed instead of silently becoming TypeScript.
- Locked Strategy Revision content/artifact immutability is enforced with a database trigger.
- Go runtime-service diagnostic redaction now recurses through arrays.
- Workshop Rust template label now says Rust alpha instead of Python experimental.
- Runtime stdout/stderr byte caps are checked independently after Wasmtime returns.

## Residual Notes

- The first ABI remains WASI Preview 1 stdin/stdout JSON. Direct exports and component model are future candidates only.
- Rust/WASM remains non-counted exhibition alpha. No counted/ranked/production promotion was made.
