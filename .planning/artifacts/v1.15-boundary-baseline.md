# v1.15 Boundary Baseline

**Captured:** 2026-05-24
**Milestone:** v1.15 Go Backend Ownership Completion

## Baseline

v1.15 starts after v1.14 promoted artifact-backed Go forks and the `strategy-runtime-abi-v1.14` boundary. Go already owns selected public reads, auth/session routes, account revision list/source/create/save, Starter/Advanced forks, and exhibition MatchSet creation when selected.

Boundary monitors start from:

- `strict_offenses=0`
- `report_only_offenses=29`

Any report-only increase is a v1.15 promotion blocker unless explicitly rebaselined with evidence.

## Go-Owned Route Baseline

Prior route ownership is recorded in `.planning/artifacts/v1.14-route-ownership-manifest.json`.

Selected Go route families at the v1.15 start:

- Public reads: public Strategy, player, ladder, MatchSet summary, replay metadata.
- Session/auth: session lookup, sign in, sign up, sign out.
- Account revisions: list, source, create/save.
- Artifact forks: Starter and Advanced fork routes backed by generated Strategy Artifact data.
- Mutation: exhibition MatchSet creation through `POST /matchsets`.

## TypeScript Lifecycle Baseline

TypeScript still owns these normal lifecycle surfaces at the v1.15 start. They are parity and rollback references for this milestone, not the future normal backend path:

- Job claim, lease, heartbeat, retry, and system failure: `packages/persistence/src/jobs.ts`.
- Coupled DB-owning worker loop: `apps/worker/src/runner.ts`.
- Match completion: `packages/persistence/src/complete-match.ts`.
- Chronicle persistence: `packages/persistence/src/chronicle-store.ts`.
- MatchSet scoring/status refresh: `packages/persistence/src/scoring.ts` and `packages/persistence/src/matchset-status.ts`.
- Public service DTO assembly and lazy scoring reads: `packages/service/src/index.ts` and `packages/persistence/src/competition.ts`.
- Persisted replay page data access: `apps/web/app/matches/server.ts`.

## Runtime Baseline

The only remaining production-ish TypeScript runtime role intended for v1.15 is isolated JS/TS Strategy execution behind `strategy-runtime-abi-v1.14`. Runtime-only TypeScript surfaces must not claim jobs, complete Matches, persist Chronicles, score MatchSets, or act as a product API fallback.

Relevant runtime code:

- `packages/spec/src/runtime.ts`
- `packages/runtime-js/src/abi-bridge.ts`
- `packages/runtime-js/src/executor.ts`

## v1.15 Ownership Direction

Go is the intended normal owner for:

- Job claim, lease, heartbeat, retry, and system-failure persistence.
- Runtime execution service handoff.
- Match lifecycle state transitions.
- Chronicle validation and persistence.
- Match completion.
- MatchSet scoring and status refresh.
- Selected public evidence delivery.
- No-fallback topology evidence.

TypeScript remains:

- `frontend` for React/web rendering and UI state.
- `runtime_only` for isolated JS/TS Strategy execution service/worker.
- `parity_only` for behavior fixtures and oracle tests.
- `rollback_only` for explicit operator rollback paths.
- `test_only` for test-support helpers.
- `deferred` for surfaces outside v1.15 scope.

## Non-Goals

v1.15 does not:

- Execute Strategy code in Go, web, or API processes.
- Use Node `vm` as a security boundary.
- Promote a production hostile-code sandbox replacement.
- Retire the TypeScript JS/TS runtime service.
- Promote counted non-JS MatchSets, ladders, or gauntlets by default.
- Move Go migration/schema ownership.
- Migrate full owner-debug replay projection.
- Migrate all workshop/admin/ladder/governance internals.
- Add Kubernetes, service mesh, cloud deployment, or production observability stack.

## Privacy Baseline

Public, service, Go, topology, monitor, and planning evidence outputs must omit actual Strategy source, StrategyMemory, SoldierMemory, objective payloads, owner debug, raw Awareness Grid, stack traces, stderr, sessions, tokens, host paths, DB DSNs, and private runtime internals by default.

Allowed source-returning exceptions remain explicit owner-private source retrieval and built-in forkable Strategy Artifact source classified by generated manifests.

## Monitor Gap To Close

Before v1.15 promotion, monitors must fail on:

- Unexpected TypeScript backend ownership creep.
- Unsafe TypeScript fallback when Go is selected.
- Mixed Go and TypeScript DB claim/completion owners.
- Lifecycle manifest drift.
- Route manifest drift.
- Runtime ABI drift.
- Schema drift.
- Privacy drift.
- Report-only offense increases.
- Public-output leaks.

The machine-readable source for these rules is `.planning/artifacts/v1.15-lifecycle-ownership-manifest.json`.
