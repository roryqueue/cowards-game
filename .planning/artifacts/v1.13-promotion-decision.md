# v1.13 Promotion Decision

**Decision:** `promote-selected-go-backend-routes`
**Date:** 2026-05-23

## Promoted

- Public reads: Strategy page, player page, ladder page, MatchSet summary, replay metadata.
- Session/auth: session read, sign-in/session create, sign-up, sign-out/session revoke.
- Account revisions: source-free revision list, owner-private source read, create/save revision.
- Mutations: exhibition MatchSet creation and TypeScript worker handoff.

## Blocked Or Deferred

- Starter and Advanced fork routes remain TypeScript-owned. Go can expose fail-closed 501 handlers, but the web does not select them by default because Go cannot yet read the library source manifests with parity.
- Match orchestration, job claiming/completion, Match execution, Chronicle generation, scoring completion, runtime failure classification, Go-owned migrations, full replay projection, owner-debug replay, production sandbox promotion, and true counted multilanguage play remain deferred.

## Evidence

- `go test ./...` passed in `apps/go-backend`.
- `pnpm test:fast` passed.
- `pnpm boundary:monitors` passed with `strict_offenses=0 report_only_offenses=29`.
- `pnpm preflight -- --skip-web` passed and completed a preflight MatchSet with public Chronicle projection.
- Live web-through-Go validation rendered Strategy, player, and MatchSet pages from Go live DB data and returned replay metadata with public-safe fields.

## Notes

The old fixture topology checker remains useful for fixture parity, but it is not the live DB proof. v1.13's live proof uses a local PostgreSQL-backed Go server and a Go-selected web process with real account, revision, exhibition, worker, and public read flows.
