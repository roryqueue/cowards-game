# v1.16 TypeScript Worker Quarantine

TypeScript worker rollback is single owner only. Stop the Go job claimant and completer, verify no Go owner is running, then start the TypeScript worker with `COWARDS_TYPESCRIPT_WORKER_PURPOSE=rollback`.

`COWARDS_MATCH_JOB_LIFECYCLE_OWNER=typescript` is not enough to start ownership. The worker also requires an explicit rollback, test, or parity purpose.

## Rollback States

| State | Procedure |
| --- | --- |
| queued_jobs | TypeScript rollback may claim only after the Go owner is stopped. |
| running_jobs | Wait for Go completion or lease expiry before rollback claim. |
| expired_leases | Reclaim only after lease expiry and single-owner verification. |
| retries | Preserve retry/failure classification. |
| exhausted_failures | Review before retry reset or manual intervention. |
| incomplete_matchsets | Scoring remains Go-owned in normal operation; rollback scoring needs an operator note. |
| scoring_public_evidence | Selected normal public summary, metadata, and evidence stay Go-owned. |

## Return To Go

Stop the TypeScript rollback worker, verify no TypeScript claims are running, then restart the Go owner. Mixed Go plus TypeScript claim or completion ownership is disallowed; an unavoidable conflict is a blocker, not an implementation path.
