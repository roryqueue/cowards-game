# v1.9 Ownership Boundary Matrix

**Milestone:** v1.9 Backend and Runtime Ownership Split  
**Created:** 2026-05-22  
**Decision:** Service-backed web read/user surfaces are the only v1.9 production ownership move.

## Active Ownership Move

| Slice | Web ownership | `@cowards/service` ownership | `@cowards/spec` ownership | Persistence ownership | Go ownership | Runtime ownership | Status |
| --- | --- | --- | --- | --- | --- | --- | --- |
| Public player profile | Page rendering, route params, UI-only shape adapters | Canonical public player profile read, DTO construction, schema parse before return | Route metadata, request/response schemas, public DTO privacy class | Internal data lookup behind service implementation only | None in v1.9 | None | Phase 58 |

## Existing Proven Boundaries

| Boundary | Current owner | v1.9 stance |
| --- | --- | --- |
| Game rules engine | `@cowards/engine` and canonical rule/spec packages | No ownership change. Engine remains pure, deterministic, serializable, and side-effect free. |
| Public MatchSet and replay metadata reads | Existing `@cowards/service` migrated slices | Preserve and use as migration pattern. |
| Service route metadata and DTO schemas | `@cowards/spec` | Remains canonical for service route ids, auth scopes, privacy classes, schemas, examples, and generated artifacts. |
| Go read-only backend | `apps/go-backend` consuming generated fixtures | No new Go route in v1.9. Future Go expansion requires service-owned DTO, TypeScript-generated parity fixtures, GET-only allowlist, topology checks, and rollback scope. |
| JS/TS Strategy execution | Worker/runtime adapters outside web/API process | No counted default change. No Strategy code executes in web/API. |
| Python/non-JS runtimes | Experimental runtime adapters and spec-owned product semantics | Remain experimental and non-counted. |

## Strict Import Enforcement Targets

Phase 57 does not add strict files. Later v1.9 phases must add each migrated page/route and its chosen dependency closure to strict import enforcement after the service slice exists.

| Phase | Expected strict target |
| --- | --- |
| Phase 58 | `apps/web/app/players/[handle]/page.tsx` and selected local closure |
| Phase 59 | Selected account page/API read files and selected local closure |
| Phase 60 | `apps/web/app/ladder/[seasonId]/page.tsx` or selected public ladder read route/page closure |

Strict targets must not import persistence roots, migrations, workers, runtime adapters, Strategy execution modules, or broad server facades such as `competitive/server` once migrated.

## v1.9 Non-Goals

- No Go mutation endpoints, auth/session mutation, persistence writes, job claiming, migrations, Match orchestration, or Strategy execution.
- No new Go read-model route in v1.9 unless the milestone is explicitly replanned.
- No production hostile-code sandbox promotion or counted Match execution through a newly promoted runtime adapter.
- No silent fallback from required runtime/topology checks to worker-thread, host subprocess, stale fixtures, or in-process execution.
- No counted Python or other non-JS MatchSets, ladders, gauntlets, analytics comparisons, or standings.
- No public language picker or package dependency ecosystem implying non-JS support parity.
- No migration of Workshop Strategy source, save, submit, validation/test execution, analytics rerun, export mutation, full replay projection, or owner-debug replay data.
- No rule, Chronicle, scoring, terminology, engine, or deterministic runtime semantic changes.

## Privacy Denylist

Public and owner-safe outputs for the v1.9 ownership move must omit Strategy source, StrategyMemory, SoldierMemory, objective payloads, owner debug, raw Awareness Grid, private Chronicle sections, stack traces, stderr, sessions, bearer tokens, host paths, environment values, and private runtime internals by default.
