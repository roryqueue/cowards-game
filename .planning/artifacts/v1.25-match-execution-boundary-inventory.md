# v1.25 Match Execution Boundary Inventory

**Date:** 2026-05-30
**Status:** Implemented

## App-Facing Contract

- `packages/spec/src/match-execution-contract.ts` defines `match-execution-app-v1`.
- App-facing DTOs: `LifecycleV1`, `MatchSetSummaryV1`, `MatchResultV1`, `ReplayMetadataV1`, `ReplayEvidenceV1`, `RuntimeEvidenceV1`, and `FailureEvidenceV1`.
- Existing `service-api-v1.8` public DTOs remain intact and are wrapped/projected into the v1.25 app-facing contract.

## Public App Consumers

- `apps/web/lib/public-service-boundary.ts` converts public MatchSet summaries into contract-backed result DTOs.
- `apps/web/app/matchsets/evidence-copy.ts` renders lifecycle state, terminality, retry disposition, result availability, replay availability, and public-safe status copy.
- `apps/web/app/matches/server.ts` can serve replay metadata/evidence through the fixture-backed contract adapter.

## Owner/Test-Only Contract

- `apps/web/lib/match-execution-fixture-adapter.ts` is explicitly gated by `PLAYWRIGHT_TEST=1`, `NODE_ENV=test`, or `COWARDS_ENABLE_MATCH_EXECUTION_FIXTURES=1`.
- The adapter returns public service DTOs sourced from schema-valid v1 fixtures, and returns `null` for unknown fixtures rather than inventing data.

## Execution-Internal Surfaces

- Runtime-service envelopes remain internal.
- No Strategy code execution moved into web/API/Go.
- No runtime promotion, production sandbox certification, direct-export ABI migration, or Component Model/WIT migration was introduced.

## Fixture Catalog

The committed contract fixture catalog covers:

- complete
- running
- queued
- strategy failure
- system failure
- timeout
- unavailable runtime
- malformed runtime result
- stale artifact
- public-safe replay

## Drift Monitors

- `scripts/check-boundary-monitors.ts` now checks the v1.25 lifecycle vocabulary, fixture scenarios, fixture adapter gates, app contract consumption, and privacy denylist markers.
