# v1.15 Promotion Decision

**Decision:** `promote-go-backend-ownership-for-selected-normal-workflows`
**Date:** 2026-05-24

## Promoted

- Go owns selected normal backend orchestration for exhibition creation, Match job lifecycle, Match completion, Chronicle persistence, MatchSet scoring, public MatchSet summary, public replay metadata, and selected public replay evidence.
- Web remains the frontend and replay renderer. When selected Go ownership is enabled, web calls Go-owned contracts and uses no silent TypeScript backend fallback.
- The TypeScript runtime execution service remains the Strategy execution boundary through `runtime-execution-service-v1.15` and `strategy-runtime-abi-v1.14`.
- Public replay evidence uses a public-only Chronicle projection and rejects owner-private projection data.
- Boundary monitors now gate v1.15 lifecycle ownership, TypeScript surface labels, failure drills, topology evidence, public-output privacy, schema drift, runtime ABI drift, and report-only import drift.

## Deferred

- TypeScript remains available as parity oracle and explicit rollback reference, not as the selected normal backend path.
- Workshop internals, admin/governance mutations, owner-debug replay, and broader ladder mutations remain deferred or test-only surfaces.
- The TypeScript runtime execution service remains the only remaining production-ish TypeScript process for Strategy execution.
- production sandbox replacement remains out of scope for v1.15.
- final TypeScript runtime retirement remains out of scope for v1.15.

## Rollback

Rollback is explicit and operator-driven:

1. Stop Go orchestration and public selected-owner traffic.
2. Switch lifecycle ownership configuration to the TypeScript rollback owner.
3. Start the TypeScript rollback worker with rollback purpose.

Go completion and TypeScript completion owners must not run against the same queue at the same time. Queued jobs, running jobs, expired leases, retries, incomplete MatchSets, and public evidence requests are covered by the rollback drill artifact.

## Privacy And Boundary Notes

Public service, Go, topology, monitor, and replay outputs remain source-safe by default. They omit strategy code, strategy memory, soldier memory, objective data, owner-private diagnostics, raw awareness data, process diagnostics, credential material, machine paths, database connection material, and runtime implementation details.
