# v1.21 Promotion Decision

Decision: **do not promote WASM/WASI, Rust, or Zig to counted, ranked, ladder, gauntlet, broad production multi-language support, or production sandbox certification.**

## Outcomes

- WASM/WASI is an executable runtime candidate lane through `runtime-wasm-wasi-wasmtime-preview1`.
- Rust is non-counted exhibition alpha only, artifact-backed by immutable `wasm32-wasip1` metadata and Wasmtime execution in runtime-service.
- Zig is fail-loud unavailable on this machine: `.planning/artifacts/v1.21-zig-readiness-evidence.json` records no local Zig toolchain.
- JS/TS remains the counted Strategy path.
- Python remains non-counted exhibition beta.
- Go remains orchestration, persistence-facing lifecycle, scoring, retry, public evidence, and promotion owner.

## Evidence

- Runtime registry and artifact schema: `packages/spec/src/runtime.ts`, `packages/spec/src/types.ts`, `packages/spec/src/schemas.ts`.
- Rust compile/artifact path: `packages/runtime-wasm-wasi/src/validation.ts`.
- Wasmtime execution path: `packages/runtime-wasm-wasi/src/wasm-wasi-subprocess-adapter.ts`.
- Runtime-service broker lane: `apps/runtime-service/src/execute-match.ts`, `apps/runtime-service/src/server.ts`.
- Go boundary validation: `apps/go-backend/runtime_service_client.go`, `apps/go-backend/live_backend.go`.

- Hostile probe evidence: `.planning/artifacts/v1.21-wasm-wasi-hostile-probe-evidence.json`.
- Zig readiness evidence: `.planning/artifacts/v1.21-zig-readiness-evidence.json`.
- Signed-in Rust proof: `.planning/artifacts/v1.21-signed-in-rust-exhibition-proof.json` and `.planning/artifacts/v1.21-signed-in-rust-exhibition-proof.md`.

## Residual Risk

This is candidate-readiness evidence, not production hostile-code isolation certification. Rust `std` currently imports WASI empty-environment functions required by the local `wasm32-wasip1` toolchain; runtime execution uses an empty Wasmtime process environment and source validation denies direct env access. Future counted promotion must replace this with deeper toolchain, ABI, sandbox, replay, privacy, rollback, abuse, and operations evidence.
