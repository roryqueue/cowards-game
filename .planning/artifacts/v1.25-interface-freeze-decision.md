# v1.25 Interface Freeze Decision

**Decision:** Frozen for parallel app/execution work.

The app-facing Match execution boundary is now versioned as `match-execution-app-v1` and covered by schema tests, fixture catalog tests, app adapter tests, result/replay rendering tests, Playwright proof, and a boundary monitor.

## Frozen Surfaces

- Lifecycle vocabulary and fields.
- App-facing MatchSet summary and Match result DTOs.
- App-facing replay metadata and replay evidence DTOs.
- Runtime evidence and failure evidence public categories.
- Public/private split for result/replay evidence.
- Fixture catalog scenarios.
- Test/dev-only fixture adapter gate.

## Intentionally Unstable

- Runtime-service internal envelopes and raw diagnostics.
- Go orchestration internals and retry implementation details.
- Owner/test-only debug DTOs.
- Future ABI candidates, including direct exports and Component Model/WIT.
- Future production sandbox claims.
- Future counted eligibility for Python, Rust, or Zig.

## Non-Claims

v1.25 does not promote runtimes, certify production sandboxing, migrate the execution ABI, expand broad ranking/ladder behavior, or allow Strategy code execution in web/API/Go.
