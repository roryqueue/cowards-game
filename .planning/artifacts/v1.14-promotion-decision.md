# v1.14 Promotion Decision

**Decision:** `promote-artifact-backed-go-forks-and-runtime-abi-v1.14`  
**Date:** 2026-05-23

## Promoted

- Generic Strategy Artifact and Strategy Revision schemas are active in `@cowards/spec`.
- Generated Strategy Artifact manifest `packages/spec/artifacts/strategy-artifacts.v1.14.json` is the parity-safe source for Starter, Advanced, and template artifacts.
- Runtime ABI `strategy-runtime-abi-v1.14` is the public request/response boundary between deterministic server orchestration and hostile Strategy code.
- Runtime JS execution now routes through the ABI bridge before adapter execution.
- Go-owned Starter and Advanced fork routes are promoted when selected, consuming generated artifacts as data only and preserving manifest source hash, validation, runtime metadata, tags, and lineage.
- Public output privacy now uses the spec-owned forbidden-field contract across service DTOs, analytics summaries, MatchSet results, and replay public projection.
- Replay preparation now rejects invalid bounds, non-canonical start layouts for canonical arenas, out-of-bounds visible pieces, visible fallen Soldiers, active/stone Soldiers without positions, overlapping Soldiers, and Soldier/terrain overlap before rendering.

## Deferred

- Go does not execute Strategy code.
- Web/API/Go backend processes do not execute hostile Strategy code.
- Node `vm` is not promoted as a security boundary.
- Strategy runtime/worker migration away from TypeScript remains deferred until there is a justified runtime migration plan.
- Static topology remains fixture-ID based, so full live browser topology is still recorded as manual smoke evidence rather than a required static monitor path.
- Production sandbox promotion and counted non-JS runtime play remain deferred.

## Privacy And Boundary Notes

Public/service/Go/topology/monitor outputs remain source-free and omit StrategyMemory, SoldierMemory, objective payloads, owner debug, raw Awareness Grid, stack traces, stderr, sessions, tokens, host paths, DB DSNs, and private runtime internals by default. The owner-private revision source endpoint remains the explicit source-returning exception.

The TypeScript service remains the parity oracle where needed. It is not the future backend path for orchestration ownership, and the runtime worker remains isolated from web/API/Go backend processes.
