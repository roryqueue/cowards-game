# v1.9 Non-JS Runtime Promotion Criteria

## Current Policy

Python and other non-JS runtimes remain experimental and non-counted in v1.9.

- Production-supported languages: JavaScript, TypeScript.
- Experimental languages: Python.
- Public language picker: disabled.
- Counted play requires explicit production support.
- Python adapter: `runtime-python-subprocess-experimental`.
- Python readiness: experimental.
- Python counted eligibility: disabled.

The canonical machine-readable policy lives in `packages/spec/src/runtime.ts` as `NON_JS_RUNTIME_SUPPORT_POLICY`, `NON_JS_RUNTIME_PROMOTION_CRITERIA`, and `assertNonJsRuntimeGuardrails`.

## Promotion Criteria

Any future non-JS counted runtime must satisfy all criteria before promotion:

| Criterion | Scope | Required Before Promotion |
| --- | --- | --- |
| Deterministic language semantics | Version, locale, hash behavior, clocks, randomness, IO, dynamic loading, memory, and output behavior | Repeated local and CI evidence proves deterministic behavior. |
| Production sandbox | Hostile-code isolation for the promoted language | Runtime isolation promotion criteria pass first. |
| Package policy | Dependency resolution, native module policy, install/build timing, supply chain, manifest and lockfile hashes | Policy is versioned and included in compatibility keys. |
| Workshop UX and docs | Templates, examples, validation, docs, support matrix, and labels | Product distinguishes production-supported and experimental languages. |
| Compatibility keys | ABI, language version, adapter version, package mode, source hash, spec, engine, capabilities, and limits | Gauntlets, ladders, replays, exports, and analytics refuse unsafe cross-runtime comparison. |
| Counted eligibility | MatchSet, ladder, gauntlet, analytics, and entry gates | All counted gates fail closed unless production support is explicit. |
| Replay/export privacy | Public replay, service, export, monitor, and topology outputs | Outputs omit private Strategy/runtime data by default. |
| Rollback policy | Disable promoted runtime safely | Existing evidence is not silently reclassified. |
| Deprecation policy | Versioned language/runtime retirement | Compatibility, replayability, and future submissions remain explainable. |

## v1.9 Guardrails

- Python stays `enabledForNormalPlay: false` and `countedResultsAllowed: false`.
- `evaluateStrategyRuntimeCountedEligibility` rejects Python with `NON_COUNTED_RUNTIME`.
- Exhibition and ladder counted gates continue rejecting Python/non-counted runtime metadata.
- Boundary monitors call `assertNonJsRuntimeGuardrails`.
- User-facing labels may say "Experimental" and "Not counted".
- No normal public language picker is introduced.
- Non-JS support does not change Strategy execution ownership, runtime isolation ownership, Go ownership, or counted standings.
