# Plan 07-03: PixiJS Replay Board and Event Callouts

<objective>
Replace the center arena placeholder with a browser-only PixiJS board renderer and pure render-model helpers that visibly distinguish Soldier states, terrain, bounds, Contraction, owner colors, numbered badges, facing, selection, and strong gameplay event callouts.
</objective>

<requirements_addressed>
VIEW-01, VIEW-05, VIEW-06
</requirements_addressed>

<threat_model>
| Threat | Severity | Mitigation |
|--------|----------|------------|
| T-07-03: Pixi canvas renders blank under Next/SSR or resize | high | Load board as a Client Component, initialize Pixi only in browser effects, use stable CSS dimensions, and add Playwright canvas-pixel checks in Plan 07-05. |
| T-07-04: Animation state mutates or drifts from canonical replay state | medium | `replay-board-model.ts` derives descriptors from selected canonical board state; animations interpolate display positions only. |
| T-07-06: Color-only visual distinctions fail legibility | medium | ACTIVE/STONE/FALLEN/TerrainStone/contraction use color plus shape/texture/badge/callout differences. |
</threat_model>
