# Plan 07-01: Replay Data Facade and Route Contracts

<objective>
Create the Match-centered replay data boundary for `/matches/{matchId}/replay`, loading persisted Chronicles by Match ID, projecting public replay data by default, exposing scoped owner/debug data explicitly, and returning replay-unavailable states without fake data.
</objective>

<requirements_addressed>
VIEW-01, VIEW-03, VIEW-04
</requirements_addressed>

<threat_model>
| Threat | Severity | Mitigation |
|--------|----------|------------|
| T-07-01: Web/API reruns Strategy code to synthesize replay data | high | Server facade only reads `createPostgresChronicleStore(pool).getByMatchId(matchId)`, `projectPublicChronicle`, `projectOwnerChronicle`, and `createReplay`; no runtime worker imports under `apps/web`. |
| T-07-02: Public replay leaks owner-only Awareness Grid, memory, objectives, source, or raw runtime details | high | Default response uses `projectPublicChronicle`; owner/debug response is explicit, scoped to one player ID, and tested with private-key string checks. |
| T-07-03: Missing or invalid Chronicle renders a misleading board | medium | Return `status: "unavailable"` with `reason: "missing-chronicle"` or `reason: "invalid-chronicle"` and render `Replay unavailable`. |
</threat_model>

  </read_first>
  <action>
    Add `apps/web/app/matches/types.ts` exporting serializable replay DTOs:
    - `ReplayViewMode = "public" | "owner"`
    - `ReplayUnavailableReason = "missing-chronicle" | "invalid-chronicle"`
    - `ReplayStatus = "ready" | "unavailable"`
    - `ReplayTimelineEntryDto` with `sequence`, `type`, `round`, `activation`, optional `cycle`, `label`, and `privacy`
    - `ReplayMetadataDto` with `matchId`, `chronicleId`, `hash`, `schemaVersion`, `eventCount`, `snapshotCount`, `outcome`, `bottomPlayerId`, `topPlayerId`, and `arenaVariantId`
    - `ReplayReadyDto` with `status: "ready"`, `mode`, `metadata`, `projection`, `timeline`, `initialSequence: 0`, and optional `ownerPlayerId`
    - `ReplayUnavailableDto` with `status: "unavailable"`, `matchId`, `reason`, and `message`
    - `ReplayPageData = ReplayReadyDto | ReplayUnavailableDto`

    Add `apps/web/app/matches/server.ts` following the dependency-injection shape from `createWorkshopServer`. It must export:
    - `createMatchReplayServer(deps?: MatchReplayServerDeps)`
    - `matchReplayServer`
    - `getMatchReplay(matchId: MatchId, options?: { mode?: ReplayViewMode; ownerPlayerId?: PlayerId }): Promise<ReplayPageData>`

    `getMatchReplay` must:
    - create and close a database pool per call through `createDatabasePool`
    - call `createPostgresChronicleStore(pool).getByMatchId(matchId)`
    - return `status: "unavailable", reason: "missing-chronicle"` when no Chronicle is stored
    - validate/reconstruct through `createReplay(projectedChronicleLikeInputOrArtifact)` or an equivalent replay-package-safe path without executing strategies
    - use `projectPublicChronicle(artifact)` when `mode` is absent or `"public"`
    - use `projectOwnerChronicle(artifact, ownerPlayerId)` only when `mode: "owner"` and `ownerPlayerId` is present
    - build `timeline` from projected events with labels such as `Match start`, `Round 1`, `Activation`, `Backstab`, `Push`, `Fall`, `Stone`, `Blocked`, `Contraction`, `Runtime violation`, and `Outcome`

    Do not import `@cowards/runtime-js/worker`, `@cowards/engine`, `apps/worker`, or Strategy execution helpers anywhere under `apps/web`.
  </action>
