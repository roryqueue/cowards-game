# Plan 07-02: Replay Client Shell, Timeline, and Inspector

<objective>
Build the React replay workbench around the data facade: a responsive page shell, draggable scrubber, step/play controls, timeline grouping, selected Soldier/event inspector, and explicit public/owner debug toggle.
</objective>

<requirements_addressed>
VIEW-02, VIEW-03, VIEW-04
</requirements_addressed>

<threat_model>
| Threat | Severity | Mitigation |
|--------|----------|------------|
| T-07-02: Client exposes private owner/debug data in public mode | high | `replay-state.ts` derives visible inspector fields from current `ReplayViewMode`; public mode labels data as `Public event` and never reads `ownerPrivate`. |
| T-07-04: Timeline state drifts from canonical replay sequence | medium | Scrubber and step controls store only selected sequence/index; derived current state is recalculated from the replay DTO/state helpers. |
| T-07-05: Responsive controls overlap board/inspector and hide replay state | medium | CSS uses stable grid zones, fixed control dimensions, wrapping controls, and viewport-specific tests. |
</threat_model>
