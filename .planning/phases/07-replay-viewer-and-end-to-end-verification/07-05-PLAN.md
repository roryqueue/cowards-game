# Plan 07-05: E2E Harness and Final Replay Verification

<objective>
Add fixture-backed and service-backed Playwright coverage for the full edit-to-replay loop, keep E2E behind a separate `pnpm e2e` command, and close Phase 7 validation with browser checks for board rendering, timeline controls, inspector behavior, Workshop handoff, and privacy defaults.
</objective>

<requirements_addressed>
TEST-06, VIEW-01, VIEW-02, VIEW-03, VIEW-04, VIEW-05, VIEW-06, VIEW-07
</requirements_addressed>

<threat_model>
| Threat | Severity | Mitigation |
|--------|----------|------------|
| T-07-10: Test-support routes expose fixture or worker execution in production | high | Routes check `NODE_ENV === "test"` or explicit test env flag and return 404/403 outside test/development. |
| T-07-11: E2E hides privacy leak by only testing owner mode | high | Fixture-backed tests assert public screenshots/DOM do not contain private markers before enabling owner/debug. |
| T-07-12: E2E becomes flaky due to a separately running worker daemon | medium | Service smoke triggers a single inline/test-mode worker execution helper rather than requiring a long-lived worker process. |
</threat_model>
