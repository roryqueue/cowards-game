# Phase 70 Plan 01: Capture v1.11 Boundary Baseline

## Objective

Record the current 30-offense broad web report-only baseline, classify every offense, name the selected Workshop read slices, and lock the non-goals before implementation begins.

## Tasks

1. Run `pnpm boundary:imports`, `pnpm boundary:monitors`, and `pnpm topology:check -- --json`.
2. Write the baseline evidence artifact with command outputs and Go/runtime/non-JS posture.
3. Write the ownership matrix classifying all 30 report-only fingerprints.
4. Identify selected slices, deferred surfaces, cleanup-only constraints, and rollback triggers.
5. Validate SCOPE-01 through SCOPE-05 against the artifacts.

## Acceptance Criteria

- `strict_offenses=0 report_only_offenses=30` is recorded as the v1.11 starting baseline.
- All 30 offenses are classified by surface and defer/selection status.
- Workshop test-summary and analytics-compare GET reads are selected.
- The selected route imports are recognized as strict-enforcement targets, while count reduction must come from real fingerprint removal.
- Go writes, production Go routing, runtime promotion, non-JS counted play, Strategy source, replay private projection, and Workshop write/runtime flows remain out of scope.
