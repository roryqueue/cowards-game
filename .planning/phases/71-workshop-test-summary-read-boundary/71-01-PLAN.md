# Phase 71 Plan 01: Move Workshop Test Summary GET Behind Service

## Objective

Make `GET /api/workshop/tests/{matchSetId}` use a spec/service-owned source-free DTO while preserving the route response shape and leaving launch/source/runtime behavior TypeScript-owned.

## Tasks

1. Add Workshop test-summary payload and service-envelope schemas to `@cowards/spec`.
2. Add matching TypeScript interfaces/exports.
3. Add `getWorkshopTestSummary(matchSetId)` to `CowardsService`, with schema parsing and privacy checks.
4. Add web Workshop read adapter/boundary modules.
5. Update the GET route to call the service boundary and return the raw-compatible summary payload.
6. Add service/spec/web route tests for success, missing summary, and private field rejection.

## Acceptance Criteria

- External API response shape is unchanged.
- Missing summary still returns `{ error: "Match set not found" }` with 404.
- DTO omits Strategy source, memories, objective payloads, owner debug, raw Awareness Grid, stack traces, stderr, sessions, tokens, host paths, and private runtime internals.
- Workshop test launch, source validation/save/retrieval, runtime execution, Match orchestration, and worker behavior are untouched.
