# Plan 03-02: Chronicle Construction and Boundary Capture

<objective>
Enrich deterministic engine summaries and implement a replay builder that produces canonical Chronicles with required events, owner-only debug sections, and boundary snapshots.
</objective>

<requirements_addressed>
REPLAY-01, REPLAY-02, REPLAY-03, REPLAY-04, REPLAY-07
</requirements_addressed>

<threat_model>
| Threat | Severity | Mitigation |
|--------|----------|------------|
| T-03-03: Awareness Grid debug data is impossible to recover after runtime invocation | high | Record the exact `SoldierBrainInput.awarenessGrid`, objective reference, owner ID, and memory debug data in deterministic private payload candidates during activation resolution. |
| T-03-04: Boundary snapshots are missing at replay-critical points | high | Add a Chronicle run builder that captures MATCH, ROUND, ACTIVATION, CONTRACTION, and terminal snapshots from actual engine states. |
| T-03-05: Construction duplicates or contradicts engine rule execution | medium | Use existing engine transition functions and parity-test final outcome against `runMatch` for the same input. |
</threat_model>

  <action>
    Update `TransitionEventSummary` so payloads may carry deterministic Chronicle construction metadata without mutating `GameState`. Add optional fields `context?: ChronicleEventContext`, `privacy?: ChroniclePrivacy`, and `privatePayload?: JsonValue`.

    In `resolveActivationSelection`, emit `STRATEGY_EVALUATED` with public payload `{ playerId }` and private payload `{ playerId, strategyMemory }` after schema parsing.

    In `resolveActivation`, when creating `input = createSoldierBrainInput(...)`, emit `AWARENESS_GRID_OBSERVED` with public payload `{ soldierId, cycleIndex }`, context containing `soldierId` and `cycleIndex`, privacy `"owner"`, and private payload containing `{ soldierId, ownerPlayerId, cycleIndex, awarenessGrid: input.awarenessGrid, objectiveRef: { hasObjective: objective !== undefined }, objectivePayload: objective }`.

    When `ACTION_EMITTED` is emitted after parsing, include private payload `{ soldierId, ownerPlayerId, soldierMemory: parsed.data.soldierMemory }`. For `RUNTIME_VIOLATION`, keep public payload category/marker safe and move raw violation message into private payload when an owning soldier/player is known.
  </action>

<task id="03-02-02" type="execute">
  <title>Implement Chronicle builder with boundary snapshots</title>
  <read_first>
    - packages/replay/src/index.ts
    - packages/engine/src/match.ts
    - packages/engine/src/activation.ts
    - packages/engine/src/contraction.ts
    - packages/engine/src/selectors.ts
    - packages/spec/src/types.ts
  </read_first>
  <action>
    Add `packages/replay/src/build.ts` exporting `buildChronicleFromMatch(input)` and `buildChronicleFromResult(args)`.

    `buildChronicleFromMatch(input)` must run the same deterministic match flow as `runMatch`, collect boundary snapshots with `getFullBoardSnapshot`, and return `{ chronicle, finalState }`. Required snapshot kinds are `MATCH_START`, `ROUND_START`, `ROUND_END`, `ACTIVATION_START`, `ACTIVATION_END`, `CONTRACTION`, and `MATCH_END` or `TERMINAL`.

    `buildChronicleFromResult(args)` must adapt an existing `{ state, events }` result into a Chronicle when only start/end snapshots are available, and mark missing intermediate snapshot validation as typed warnings/errors handled by Plan 03-03.

    Chronicle construction must:
    - preserve global event `sequence`
    - preserve phase/round/activation/cycle context where available
    - copy private payloads into `private.byPlayerId[playerId]` entries keyed by deterministic references such as `private:event:{sequence}`
    - store public events with `privateRef` instead of private grids/objectives/memory
    - fill reproducibility metadata from match input versions, seed, arena variant, strategy revisions, and match ID
  </action>
  <verify>
    pnpm --filter @cowards/replay test -- build.test.ts
    rg "export const buildChronicleFromMatch" packages/replay/src/build.ts
    rg "MATCH_START" packages/replay/src/build.ts
    rg "private:event" packages/replay/src/build.ts
  </verify>
  <acceptance_criteria>
    - `packages/replay/src/build.ts` contains `export const buildChronicleFromMatch`
    - `packages/replay/src/build.ts` contains `MATCH_START`
    - `packages/replay/src/build.ts` contains `ACTIVATION_END`
    - `packages/replay/src/build.ts` contains `private:event`
    - `pnpm --filter @cowards/replay test -- build.test.ts` exits 0

</task>
