# Plan 06-03: Revision Submission, History, and Selection

<objective>
Complete the immutable Strategy Revision loop: submit valid drafts with label/notes metadata, reject invalid drafts, browse persisted revision history, select revisions, and load selected revision source back into an editable draft copy.
</objective>

<requirements_addressed>
UX-04, UX-06
</requirements_addressed>

<threat_model>
| Threat | Severity | Mitigation |
|--------|----------|------------|
| T-06-09: UI mutates an immutable Strategy Revision by editing loaded history source | high | Loading a revision copies its source into the draft editor and never updates the existing revision row. |
| T-06-10: Invalid source is persisted through client-side bypass | high | Server submission rebuilds and validates Strategy Revision and rejects invalid reports regardless of client state. |
| T-06-11: Revision history exposes irrelevant opponent source or private runtime payloads | medium | History API lists only the local Workshop strategy; opponent source stays out of revision history DTOs. |
</threat_model>
