# Plan 06-04: Workshop Test Match Launch and Status

<objective>
Enable users to launch quick local Workshop test MatchSets from selected revisions against bundled sample opponents, then view honest pending/running/complete/degraded/failed status and scoring summaries without implementing the Phase 7 replay UI.
</objective>

<requirements_addressed>
UX-05, UX-06
</requirements_addressed>

<threat_model>
| Threat | Severity | Mitigation |
|--------|----------|------------|
| T-06-12: Workshop test launch runs Strategy code inside the web/API process | high | Launch persisted MatchSets only; rely on worker/job path for execution. Verify `apps/web` does not import runtime worker or engine/replay execution APIs. |
| T-06-13: UI pretends queued jobs are completed when no worker is running | medium | Display exact MatchSet status returned by persistence and provide refresh/polling. |
| T-06-14: Test UI exposes replay-private Strategy source or memory payloads | high | Show only MatchSet IDs, statuses, match count, and aggregate scoring; no Chronicle/private data rendering. |
</threat_model>
