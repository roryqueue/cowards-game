# Plan 06-02: Monaco Workbench, Templates, and Live Validation

<objective>
Replace the scaffold homepage with the first Strategy Workshop screen: a single focused editor workbench with bundled templates, Monaco editing, live debounced validation, manual validation, and actionable validation results.
</objective>

<requirements_addressed>
UX-01, UX-02, UX-03
</requirements_addressed>

<threat_model>
| Threat | Severity | Mitigation |
|--------|----------|------------|
| T-06-06: Monaco import breaks server rendering or App Router builds | high | Isolate Monaco in a Client Component loaded through `next/dynamic` with `ssr: false`; keep server shell props serializable. |
| T-06-07: Validation UX implies line-level/typechecker precision that runtime validation does not provide | medium | Display source-level validation rows only; do not show fake line diagnostics. |
| T-06-08: User loses draft source by applying a template accidentally | medium | Track dirty source and require a replace-draft confirmation before overwriting edited source with a template. |
</threat_model>
