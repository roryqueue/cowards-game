# Plan 06-05: Workshop UX Hardening and Browser Verification

<objective>
Harden the completed Workshop against layout, accessibility, responsive, and browser-rendering failures; verify the full edit -> validate -> submit -> select -> launch test loop without crossing into Phase 7 replay scope.
</objective>

<requirements_addressed>
UX-01, UX-02, UX-03, UX-04, UX-05, UX-06
</requirements_addressed>

<threat_model>
| Threat | Severity | Mitigation |
|--------|----------|------------|
| T-06-15: Monaco or responsive panels render blank, overlap, or become unusable on common viewports | high | Run browser checks at desktop, laptop, tablet, and mobile viewports; adjust CSS with stable dimensions and wrapping rules. |
| T-06-16: Color-only status indicators make validation/test status inaccessible | medium | Ensure status text labels accompany all color states and validation/test panels use accessible status regions. |
| T-06-17: Final UI drifts beyond Phase 6 into replay or generic strategy management | medium | Verify no replay board/timeline or multi-strategy management UI appears; keep first screen Workshop-only. |
</threat_model>

<task id="06-05-01" type="execute">
  <title>Polish responsive workbench layout and accessibility states</title>
  <read_first>
    - apps/web/app/globals.css
    - apps/web/app/workshop/workshop-client.tsx

    - .planning/phases/06-strategy-workshop-ux/06-VALIDATION.md
  </read_first>
  <action>
    Review and update `globals.css` and `workshop-client.tsx` so the final UI satisfies all UI-SPEC safety rules:
    - desktop three-zone grid with 260px left rail, `minmax(520px, 1fr)` center, and 340px right rail at `>= 1180px`
    - tablet layout below `1180px` with editor full width and lower panels in two columns
    - mobile layout below `760px` with single-column order: header, template selector, Monaco editor, validation, submit revision, revision history, test launch/status
    - Monaco minimum heights: 520px desktop, 460px tablet, 360px mobile
    - no horizontal overflow at 360px viewport width
    - all input controls have visible labels
    - validation summary uses `role="status"` or `aria-live="polite"`
    - disabled submit/test buttons have nearby explanatory text
    - focus ring remains `2px solid #256D85`
    - color states include text labels `Valid`, `Invalid`, `Pending`, `Running`, `Complete`, or `Failed`

    Remove any marketing/hero copy, generic programming tutorial text, replay board/timeline UI, and multi-strategy management UI that may have slipped in.
  </action>
  <verify>
