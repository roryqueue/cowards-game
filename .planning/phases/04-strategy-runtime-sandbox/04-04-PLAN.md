# Plan 04-04: Runtime Integration, Documentation, and Verification Closure

<objective>
Prove the JS runtime works through the engine and Chronicle pipeline, document the prototype sandbox boundary, polish exports, and close Phase 4 validation coverage.
</objective>

| Threat | Severity | Mitigation |
|--------|----------|------------|
| T-04-11: Runtime tests pass in isolation but fail through engine execution | high | Add integration tests that run engine/replay with `createRuntimeFromRevision`. |

  <action>
    Add `packages/runtime-js/src/integration.test.ts` proving:
    - a valid default-object strategy revision can be built, converted to a runtime with `createRuntimeFromRevision`, and used by `runMatch`
    - `selectActivations` receives full-board `StrategyInput` and returns activation orders plus StrategyMemory
    - `soldierBrain` receives local `SoldierBrainInput` and returns exactly one Action plus SoldierMemory
    - a thrown `soldierBrain` exception produces a `RUNTIME_VIOLATION` event in a built Chronicle
    - public projection includes the runtime violation type marker and context but not raw exception text
    - owner projection includes the raw exception detail for the owning player
    - invalid output causes the same gameplay result as current engine semantics: interrupted Activation and no-advance stoning if the Soldier did not Advance
  </action>
