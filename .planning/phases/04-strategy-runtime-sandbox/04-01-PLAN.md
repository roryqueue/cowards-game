# Plan 04-01: Strategy Revision Contracts and Safe API Surface

<objective>
Define canonical Strategy Revision contracts in `@cowards/spec` and expose the safe, non-executable `@cowards/runtime-js` API surface for validation and artifact construction.
</objective>

<requirements_addressed>
RUN-01, RUN-02, RUN-05, RUN-10
</requirements_addressed>

<threat_model>
| Threat | Severity | Mitigation |
|--------|----------|------------|
| T-04-01: Strategy Revision metadata is runtime-local and cannot be reused by persistence or UX | high | Add Strategy Revision types and schemas to `packages/spec`, not `packages/runtime-js`. |
| T-04-02: Revision identity depends on clock/random values | high | Require deterministic content-derived IDs and hashes in the contract and schema tests. |
| T-04-03: Executable runtime leaks through the default package import | medium | Keep `@cowards/runtime-js` as validation/artifact API only; executable worker entrypoint is added later as a separate export. |
</threat_model>

<tasks>
<task id="04-01-01" type="execute">
  <title>Add canonical Strategy Revision types</title>
  <read_first>
    - packages/spec/src/types.ts
    - packages/spec/src/schemas.ts
    - packages/spec/src/constants.ts
    - packages/spec/src/versions.ts
    - .planning/phases/04-strategy-runtime-sandbox/04-CONTEXT.md
    - .planning/phases/04-strategy-runtime-sandbox/04-RESEARCH.md
  </read_first>
  <action>
    Extend `packages/spec/src/types.ts` with these exact exported contracts:
    - `export type StrategyRuntimeName = "runtime-js"`
    - `export type StrategyRevisionValidationSeverity = "error" | "warning"`
    - `export type StrategyRevisionValidationCode = "SOURCE_TOO_LARGE" | "FORBIDDEN_PATTERN" | "MISSING_DEFAULT_EXPORT" | "MISSING_SELECT_ACTIVATIONS" | "MISSING_SOLDIER_BRAIN" | "ASYNC_METHOD_NOT_ALLOWED" | "IMPORT_NOT_ALLOWED" | "TRANSPILE_FAILED" | "ENGINE_INCOMPATIBLE"`
    - `export interface StrategyRevisionValidationIssue { code: StrategyRevisionValidationCode; severity: StrategyRevisionValidationSeverity; message: string; pattern?: string | undefined }`
    - `export interface StrategyRevisionValidationReport { valid: boolean; errors: StrategyRevisionValidationIssue[]; warnings: StrategyRevisionValidationIssue[]; sourceBytes: number; forbiddenPatterns: string[]; sourceHash: string; runtimeVersion: string; engineCompatibility: { spec: string; engine: string } }`
    - `export interface StrategyRevisionMetadata { createdBy?: string | undefined; label?: string | undefined; tags?: string[] | undefined }`
    - `export interface StrategyRevision { id: StrategyRevisionId; strategyId?: StrategyId | undefined; source: string; sourceHash: string; sourceBytes: number; runtime: { name: StrategyRuntimeName; version: string }; engineCompatibility: { spec: string; engine: string }; validation: StrategyRevisionValidationReport; metadata: StrategyRevisionMetadata }`

    The contract must keep raw `source` plus `sourceHash`, and must not include persistence/database fields, wall-clock timestamps, random IDs, queue state, or match execution state.
  </action>
  <verify>
    pnpm --filter @cowards/spec typecheck
    rg "export interface StrategyRevision " packages/spec/src/types.ts
    rg "StrategyRevisionValidationReport" packages/spec/src/types.ts
    rg "StrategyRuntimeName = \"runtime-js\"" packages/spec/src/types.ts
  </verify>
  <acceptance_criteria>
    - `packages/spec/src/types.ts` contains `export interface StrategyRevision`
    - `packages/spec/src/types.ts` contains `export interface StrategyRevisionValidationReport`
    - `packages/spec/src/types.ts` contains `StrategyRuntimeName = "runtime-js"`
    - `packages/spec/src/types.ts` does not contain `createdAt`
    - `pnpm --filter @cowards/spec typecheck` exits 0
  </acceptance_criteria>
</task>

<task id="04-01-02" type="test">
  <title>Add Strategy Revision schemas and schema tests</title>
  <read_first>
    - packages/spec/src/types.ts
    - packages/spec/src/schemas.ts
    - packages/spec/src/spec.test.ts
    - packages/spec/src/constants.ts
  </read_first>
  <action>
    Extend `packages/spec/src/schemas.ts` with:
    - `StrategyRuntimeNameSchema = z.literal("runtime-js")`
    - `StrategyRevisionValidationSeveritySchema`
    - `StrategyRevisionValidationCodeSchema`
    - `StrategyRevisionValidationIssueSchema`
    - `StrategyRevisionValidationReportSchema`
    - `StrategyRevisionMetadataSchema`
    - `StrategyRevisionSchema`

    `StrategyRevisionSchema` must enforce:
    - `id` is a non-empty string
    - `source` is a non-empty string whose byte length is `<= STRATEGY_SOURCE_BYTES`
    - `sourceHash` is a non-empty string
    - `sourceBytes` is an integer from `0` through `STRATEGY_SOURCE_BYTES`
    - `runtime.name` is exactly `"runtime-js"`
    - `runtime.version` is a non-empty string
    - `engineCompatibility.spec` and `engineCompatibility.engine` are non-empty strings
    - `validation.valid === (validation.errors.length === 0)` using a schema refinement

    Update `packages/spec/src/spec.test.ts` with:
    - a valid Strategy Revision fixture that parses through `StrategyRevisionSchema`
    - an invalid oversized source fixture that fails `StrategyRevisionSchema.safeParse`
    - an invalid report where `valid: true` and `errors` is non-empty, which fails schema parsing
  </action>
  <verify>
    pnpm --filter @cowards/spec test -- spec.test.ts
    rg "export const StrategyRevisionSchema" packages/spec/src/schemas.ts
    rg "StrategyRevisionSchema.parse" packages/spec/src/spec.test.ts
  </verify>
  <acceptance_criteria>
    - `packages/spec/src/schemas.ts` contains `export const StrategyRevisionSchema`
    - `packages/spec/src/schemas.ts` contains `STRATEGY_SOURCE_BYTES`
    - `packages/spec/src/spec.test.ts` contains `StrategyRevisionSchema.parse`
    - `packages/spec/src/spec.test.ts` contains `valid: true`
    - `pnpm --filter @cowards/spec test -- spec.test.ts` exits 0
  </acceptance_criteria>
</task>
