# Plan 04-02: Validation, Hashing, Transpilation, and Immutable Revisions

<objective>
Implement the safe `@cowards/runtime-js` APIs that validate JS/TS strategy source, detect forbidden capabilities, transpile TypeScript syntax, derive deterministic hashes/IDs, and build immutable Strategy Revision artifacts.
</objective>

<requirements_addressed>
RUN-01, RUN-02, RUN-05, RUN-08, RUN-10, TEST-04
</requirements_addressed>

<threat_model>
| Threat | Severity | Mitigation |
|--------|----------|------------|
| T-04-04: Forbidden capability code is accepted as a valid revision | high | Add static source scanning for imports, dynamic imports, `eval`, `Function`, worker spawning, process/env/fs/network APIs, clocks, randomness, native modules, and package installation. |
| T-04-05: Revision IDs vary across machines or runs | high | Hash normalized deterministic source/version inputs with `node:crypto` and no wall-clock/random data. |
| T-04-06: TypeScript transpilation is mistaken for full typechecking | medium | Name and document the API as transpile-only and add validation warnings for unsupported single-file constructs when detectable. |
</threat_model>

<task id="04-02-02" type="test">
  <title>Implement source validation and forbidden pattern tests</title>
  <read_first>
    - packages/runtime-js/src/index.ts
    - packages/spec/src/constants.ts
    - packages/spec/src/schemas.ts
    - .planning/phases/04-strategy-runtime-sandbox/04-CONTEXT.md
  </read_first>
  <action>
    Add `packages/runtime-js/src/validation.ts` exporting:
    - `FORBIDDEN_SOURCE_PATTERNS`
    - `validateStrategySource(source: string, options?: { runtimeVersion?: string; specVersion?: string; engineVersion?: string }): StrategyRevisionValidationReport`

    `FORBIDDEN_SOURCE_PATTERNS` must include entries that detect:
    - static imports: `import ... from`, `import "module"`
    - dynamic imports: `import(`
    - `eval(`
    - `Function(` and `new Function`
    - `process`, `process.env`
    - `require(`
    - `node:fs`, `fs/promises`, `node:http`, `node:https`, `fetch(`
    - `Date.now`, `new Date`
    - `Math.random`
    - `Worker(`, `worker_threads`, `child_process`
    - `WebAssembly`
    - package install tokens: `npm install`, `pnpm add`, `yarn add`

    `validateStrategySource` must:
    - compute `sourceBytes` with `jsonByteLength(source)` or equivalent UTF-8 byte count
    - return `SOURCE_TOO_LARGE` error when `sourceBytes > STRATEGY_SOURCE_BYTES`
    - return `FORBIDDEN_PATTERN`, `IMPORT_NOT_ALLOWED`, or `ASYNC_METHOD_NOT_ALLOWED` errors as appropriate
    - require source to contain `export default`
    - require `selectActivations` and `soldierBrain`
    - treat `async selectActivations` and `async soldierBrain` as errors
    - fill `runtimeVersion`, `engineCompatibility`, `sourceHash`, `errors`, `warnings`, `forbiddenPatterns`, and `valid`

    Add `packages/runtime-js/src/validation.test.ts` covering every forbidden category above, source size overflow, missing default export, missing method names, async method rejection, and a valid minimal strategy.
  </action>
  <verify>
    pnpm --filter @cowards/runtime-js test -- validation.test.ts
    rg "FORBIDDEN_SOURCE_PATTERNS" packages/runtime-js/src/validation.ts
    rg "ASYNC_METHOD_NOT_ALLOWED" packages/runtime-js/src/validation.test.ts
  </verify>
  <acceptance_criteria>
    - `packages/runtime-js/src/validation.ts` contains `export const FORBIDDEN_SOURCE_PATTERNS`
    - `packages/runtime-js/src/validation.ts` contains `export const validateStrategySource`
    - `packages/runtime-js/src/validation.test.ts` contains `Math.random`
    - `packages/runtime-js/src/validation.test.ts` contains `ASYNC_METHOD_NOT_ALLOWED`
    - `pnpm --filter @cowards/runtime-js test -- validation.test.ts` exits 0
  </acceptance_criteria>
</task>

    Add `packages/runtime-js/src/transpile.ts` exporting:
    - `transpileStrategySource(source: string): { ok: true; code: string } | { ok: false; message: string }`

    Use TypeScript's transpile-only API with options:
    - `module: ts.ModuleKind.ESNext`
    - `target: ts.ScriptTarget.ES2022`
    - `isolatedModules: true`
    - no full typecheck/program creation

    The transpiled code must preserve the `export default` strategy object shape for later worker evaluation.

    Add `packages/runtime-js/src/transpile.test.ts` proving:
    - a source string with TypeScript parameter annotations transpiles
    - the transpiled code contains `export default`
    - no full typecheck is required for a structurally valid source string
    - syntax failures return `{ ok: false }`
  </action>

<task id="04-02-04" type="execute">
  <title>Implement immutable Strategy Revision builder</title>
  <read_first>
    - packages/runtime-js/src/hash.ts
    - packages/runtime-js/src/validation.ts
    - packages/runtime-js/src/transpile.ts
    - packages/spec/src/schemas.ts

  <action>
    Add `packages/runtime-js/src/revision.ts` exporting:
    - `buildStrategyRevision(input: { source: string; strategyId?: string | undefined; metadata?: StrategyRevisionMetadata | undefined }): StrategyRevision`
    - `isValidStrategyRevision(revision: unknown): boolean`

    `buildStrategyRevision` must:
    - call `validateStrategySource(source)`
    - compute `sourceHash` and content-derived `id`
    - set `runtime: { name: "runtime-js", version: COMPATIBILITY_VERSIONS.runtimeJs }`
    - set `engineCompatibility: { spec: COMPATIBILITY_VERSIONS.spec, engine: COMPATIBILITY_VERSIONS.engine }`
    - include raw `source`, `sourceHash`, `sourceBytes`, `validation`, and provided metadata
    - parse the artifact with `StrategyRevisionSchema`
    - return a deeply frozen object or at minimum freeze the top-level object plus nested `runtime`, `engineCompatibility`, `validation`, `metadata`, `errors`, and `warnings`

    Export builder APIs from `packages/runtime-js/src/index.ts`.

    Add `packages/runtime-js/src/revision.test.ts` proving:
    - two identical inputs produce identical IDs
    - changing source changes ID
    - returned artifact stores raw source plus source hash
    - artifact parses through `StrategyRevisionSchema`
    - returned artifact is frozen
    - invalid source returns an artifact with `validation.valid === false` without throwing for normal validation failures
  </action>
