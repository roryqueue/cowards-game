# Phase 74 Plan 01: Capture Required Live Go Evidence

## Objective

Make live local Go topology evidence a required validation lane while keeping Go read-only, fixture-backed, and out of production web routing.

## Tasks

1. Run `pnpm go:parity`.
2. Run `pnpm boundary:monitors`.
3. Start the local Go backend separately from `apps/go-backend`.
4. Run `pnpm topology:check -- --require-go --json`.
5. Record sanitized evidence and rollback/non-promotion notes.
6. Confirm required Go failures are required failures, not silent TypeScript fallback.

## Acceptance Criteria

- Live topology passes for Go health, public MatchSet, public replay metadata, public Strategy page, unauthenticated owner analytics rejection, and diagnostic privacy.
- Go route inventory remains GET-only and fixture-backed.
- Go writes, auth/session mutation, ladder writes, Match orchestration, jobs, migrations, persistence ownership, Strategy source retrieval, Strategy execution, production web routing, runtime sandbox promotion, and counted non-JS play remain absent.
