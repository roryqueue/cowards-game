# Plan 09-02: Public Projection Privacy Hard Gate

## Objective

Turn public replay projection privacy into a hard package and web fixture gate.

Purpose: This implements GRAM-07 and the private-leaking portion of GRAM-08 while honoring D-07, D-08, and D-09. Projection privacy stays in `packages/replay`; web tests consume the projection output and do not re-filter raw Chronicles.

Output: Hostile privacy marker tests and any sanitizer tightening needed to keep public replay output private by default.
