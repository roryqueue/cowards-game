# Plan 09-01: Validation Error Contract and Compatibility Gate

## Objective

Establish the stable validation error-code contract and strict compatibility gate that all later Chronicle grammar checks use.

Purpose: This implements GRAM-01 and GRAM-06 while honoring D-02, D-05, D-06, D-10, and D-11. It keeps semantic validation in `packages/replay` after Zod parsing, and makes unsupported versions fail closed before replay rendering.

Output: Expanded validation error contracts, compatibility matrix tests, and replay-loading failure diagnostics.

| T-09-01-02 | Repudiation | validation errors | mitigate | Return stable error codes with clear messages; do not expose raw Zod issue text as the primary failure surface. |
