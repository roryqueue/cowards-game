# Phase 37: Demo and Regression Verification - Context

**Gathered:** 2026-05-20
**Status:** Ready for planning

<domain>
## Phase Boundary

Phase 37 performs the final v1.5 verification pass across automated tests, browser-visible demo flows, privacy/runtime isolation guarantees, determinism guarantees, and documentation updates. It should verify the actual Advanced Library, generated example MatchSets, completed demo tournament, Strategy cards, player/profile pages where applicable, and representative replays. It should not introduce new features or tune Strategies except to fix verification failures by routing work back to the owning phase.

</domain>

<decisions>
## Implementation Decisions

### Automated Regression Gates

- **D-01:** Public DTO/page tests must fail if they expose Strategy source, StrategyMemory, SoldierMemory, objective payloads, owner debug, raw Awareness Grid details, stack traces, or private runtime internals by default.
- **D-02:** Determinism tests must prove identical gauntlet profiles expand to identical matrix order, profile hash, scoring projection, and summary ordering.
- **D-03:** Runtime/web isolation tests must prove Workshop gauntlet and tournament creation routes create jobs but do not execute Strategy code in web/API processes.
- **D-04:** Privacy, determinism, runtime isolation, and canonical rule-version failures are hard blockers for milestone completion.

### Browser Verification

- **D-05:** Browser verification should cover the Advanced Library entry point, representative Advanced Strategy cards, example MatchSet result pages, the completed tournament page, player/profile pages where applicable, and representative replays.
- **D-06:** Browser verification must use realistic completed v1.5 demo data generated by prior phases, not only empty/scaffold states.
- **D-07:** Representative browser checks should confirm links connect across Advanced Library, cards, MatchSets, tournament, profiles, and replays without exposing private data.
- **D-08:** Verification should record local links and concise observations in a phase verification summary.

### Documentation and Demo Report

- **D-09:** Documentation updates are required before milestone completion.
- **D-10:** Docs should cover Workshop power tools, Advanced Library purpose, evidence vocabulary, local demo/report regeneration, example MatchSet links, tournament links, representative replay links, and non-durable demo framing.
- **D-11:** The local demo/report should be treated as the canonical human-facing summary of Advanced set performance against v1.4 starters and itself.
- **D-12:** If Phase 37 finds stale links, stale evidence, missing docs, or degenerate demo data, it should route fixes to the owning phase and rerun the relevant verification.

### the agent's Discretion

The planner may choose exact test commands, browser viewport coverage, verification summary filename, and documentation locations. It should prefer existing project test patterns and keep the final verification concise but reproducible.

</decisions>
