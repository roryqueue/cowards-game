# Plan 05-02: Match and MatchSet Creation Services

<objective>
Implement internal service APIs for creating locked Matches and MatchSets from immutable Strategy Revisions, including preset-generated concrete matrices, curated arenas, fixed seed lists, explicit side assignments, and status visibility.
</objective>

<requirements_addressed>
MATCH-01, MATCH-02, MATCH-03, MATCH-07, DATA-01, DATA-02, DATA-04
</requirements_addressed>

<threat_model>
| Threat | Severity | Mitigation |
|--------|----------|------------|
| T-05-04: A Match is created with inputs that can change before execution | high | Creation service locks StrategyRevision IDs, seed, Arena Variant, and side assignment in `matches` immediately. |
| T-05-05: Preset MatchSet regeneration changes historical evaluation meaning | high | Persist the generated concrete matrix in `match_sets.matrix` and use membership rows as canonical truth. |
| T-05-06: Dev shortcuts bypass reproducibility rules | medium | Source shortcuts must build StrategyRevisions first, then call the same ID-based creation service. |
</threat_model>

<task id="05-02-02" type="test">
  <title>Implement locked Match creation service</title>
  <read_first>
    - packages/persistence/src/repositories.ts
    - packages/persistence/src/schema.ts
    - packages/engine/src/types.ts
    - packages/spec/src/types.ts
    - .planning/phases/05-match-orchestration-and-persistence/05-CONTEXT.md

  <action>
    Add `packages/persistence/src/match-service.ts` exporting:
    - `export interface CreateMatchInput { id: MatchId; bottomStrategyRevisionId: StrategyRevisionId; topStrategyRevisionId: StrategyRevisionId; arenaVariantId: ArenaVariantId; seed: string; bottomPlayerId: PlayerId; topPlayerId: PlayerId }`
    - `export const createMatchService = (pool: Pool) => ({ createMatch, getMatchStatus })`

    `createMatch(input)` must:
    - require non-empty `seed`
    - require explicit `bottomStrategyRevisionId`, `topStrategyRevisionId`, `arenaVariantId`, `bottomPlayerId`, and `topPlayerId`
    - read both StrategyRevisions and the ArenaVariant before insert
    - call `lockStrategyRevision` for both revisions
    - insert the `matches` row with status `'pending'`
    - insert one `match_jobs` row with status `'queued'`, `attempts = 0`, and `max_attempts = DEFAULT_MAX_JOB_ATTEMPTS`
    - return the persisted Match and job ids

    `getMatchStatus(id)` must return one of `'pending'`, `'running'`, `'complete'`, `'failed_system'`, or `'blocked'`.

    Add `packages/persistence/src/match-service.test.ts` proving:
    - Match creation stores seed and side assignment exactly as supplied
    - both StrategyRevisions get `locked_at`
    - a queued job is created
    - missing seed throws
    - status `'pending'` is visible after creation
  </action>
