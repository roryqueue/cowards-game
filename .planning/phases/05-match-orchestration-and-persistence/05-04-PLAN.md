# Plan 05-04: Worker Job Claiming, Execution, Retry, and Idempotent Completion

<objective>
Implement the worker orchestration path that claims leased Match jobs, executes deterministic simulations, stores Chronicles and outcomes durably, records attempts, retries system failures, and keeps strategy runtime violations classified as gameplay events.
</objective>

<requirements_addressed>
MATCH-04, MATCH-05, MATCH-07, TEST-05
</requirements_addressed>

<threat_model>
| Threat | Severity | Mitigation |
|--------|----------|------------|
| T-05-10: Two workers complete the same Match and corrupt replay state | high | Use lease tokens, status guards, and unique Chronicle per Match for idempotent completion. |
| T-05-11: Strategy runtime violation is misclassified as infrastructure failure | high | Tests must prove runtime violations complete through engine/replay and do not increment system failure retry classification. |
| T-05-12: Worker crash leaves jobs stuck forever | high | Lease expiration and reclaim query must recover running jobs whose lease expired. |
| T-05-13: Chronicle persistence failure marks Match complete anyway | high | Completion transaction must only mark Match/job complete after Chronicle storage succeeds. |
</threat_model>

<task id="05-04-01" type="execute">
  <title>Implement leased job claiming and attempt records</title>
  <read_first>
    - packages/persistence/migrations/0001_initial.sql
    - packages/persistence/src/schema.ts
    - packages/persistence/src/db.ts
    - .planning/phases/05-match-orchestration-and-persistence/05-CONTEXT.md
  </read_first>
  <action>
    Add `packages/persistence/src/jobs.ts` exporting:
    - `export interface ClaimMatchJobInput { workerId: string; leaseMs?: number; now?: Date }`
    - `export interface ClaimedMatchJob { jobId: string; matchId: MatchId; attemptNumber: number; leaseToken: string; leaseExpiresAt: Date }`
    - `export const DEFAULT_LEASE_MS = 30_000`
    - `export const claimNextMatchJob = async (pool: Pool, input: ClaimMatchJobInput): Promise<ClaimedMatchJob | null>`
    - `export const heartbeatMatchJob = async (pool: Pool, input: { jobId: string; leaseToken: string; leaseMs?: number }): Promise<boolean>`
    - `export const recordAttemptFailure = async (pool: Pool, input: { jobId: string; leaseToken: string; errorClass: string; errorMessage: string; retryable: boolean; details?: JsonValue }): Promise<"retry_queued" | "failed_system">`

    `claimNextMatchJob` must use SQL with `FOR UPDATE SKIP LOCKED` and claim jobs where:
    - `status = 'queued' and run_after <= now()`, or
    - `status = 'running' and lease_expires_at < now()`

    It must:
    - set `status = 'running'`
    - set `worker_id`
    - generate a unique `lease_token`
    - set `lease_expires_at`
    - increment `attempts`
    - insert a `match_job_attempts` row with `status = 'running'`
    - update the related Match status to `'running'`

    `recordAttemptFailure` must:
    - write `finished_at`, `status = 'failed_system'`, error class/message/details, and retryable flag on the attempt
    - requeue the job when retryable and attempts remain
    - mark both job and Match `failed_system` when attempts are exhausted
  </action>
  <verify>
    pnpm --filter @cowards/persistence test -- jobs.test.ts
    rg "FOR UPDATE SKIP LOCKED" packages/persistence/src/jobs.ts
    rg "lease_token" packages/persistence/src/jobs.ts
    rg "failed_system" packages/persistence/src/jobs.ts
  </verify>
  <acceptance_criteria>
    - `packages/persistence/src/jobs.ts` contains `claimNextMatchJob`
    - `packages/persistence/src/jobs.ts` contains `FOR UPDATE SKIP LOCKED`
    - `packages/persistence/src/jobs.ts` contains `heartbeatMatchJob`
    - `packages/persistence/src/jobs.ts` contains `recordAttemptFailure`
    - `pnpm --filter @cowards/persistence test -- jobs.test.ts` exits 0

    - behave idempotently if the Match is already complete with an existing Chronicle
