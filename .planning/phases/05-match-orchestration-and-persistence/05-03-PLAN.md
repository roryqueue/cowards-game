# Plan 05-03: Chronicle Storage Adapter and Metadata Persistence

<objective>
Add a Chronicle storage adapter with a PostgreSQL JSONB implementation that stores unified full Chronicle artifacts and replay/query metadata while enforcing one durable Chronicle per Match.
</objective>

<requirements_addressed>
MATCH-04, DATA-01, DATA-03
</requirements_addressed>

<threat_model>
| Threat | Severity | Mitigation |
|--------|----------|------------|
| T-05-07: Match completes without a replayable Chronicle | high | Completion service must call Chronicle storage before marking Match/job complete. |
| T-05-08: Private Chronicle sections leak by being stripped into public storage incorrectly | high | Store the unified logical Chronicle artifact; public/private access remains a projection concern. |
| T-05-09: Duplicate worker completion creates multiple Chronicles for one Match | high | Enforce unique `chronicles.match_id` and expose idempotent store/read behavior. |
</threat_model>

<tasks>
<task id="05-03-01" type="execute">
  <title>Implement Chronicle storage adapter contract</title>
  <read_first>
    - packages/replay/src/build.ts
    - packages/replay/src/validate.ts
    - packages/replay/src/hash.ts
    - packages/spec/src/types.ts
    - packages/persistence/migrations/0001_initial.sql
    - .planning/phases/05-match-orchestration-and-persistence/05-CONTEXT.md
  </read_first>
  <action>
    Add `packages/persistence/src/chronicle-store.ts` exporting:
    - `export interface ChronicleMetadata { id: string; matchId: MatchId; schemaVersion: string; hash: string; outcome: JsonValue; eventCount: number; snapshotCount: number; bottomPlayerId: PlayerId; topPlayerId: PlayerId; bottomStrategyRevisionId: StrategyRevisionId; topStrategyRevisionId: StrategyRevisionId; arenaVariantId: ArenaVariantId }`
    - `export interface StoredChronicle { metadata: ChronicleMetadata; artifact: Chronicle }`
    - `export interface ChronicleStore { put(chronicle: Chronicle): Promise<StoredChronicle>; getByMatchId(matchId: MatchId): Promise<StoredChronicle | null> }`
    - `export const createPostgresChronicleStore = (pool: Pool): ChronicleStore`

    `put(chronicle)` must:
    - validate with `validateChronicle(chronicle)`
    - throw `ChronicleValidationSystemFailure` or an exported error class when validation fails
    - compute hash with `createChronicleContentHash` or existing replay hash helper
    - derive `id` as `chronicle:${hash}`
    - extract metadata from `chronicle.reproducibility`, `chronicle.events`, and `chronicle.snapshots`
    - insert full `artifact` into `chronicles.artifact` JSONB
    - preserve `chronicle.private` if present
    - use `on conflict (match_id) do nothing` or equivalent idempotency and return the existing row when one already exists for the Match

    Export the adapter from `packages/persistence/src/index.ts`.
  </action>
  <verify>

    rg "ChronicleStore" packages/persistence/src/chronicle-store.ts
    rg "validateChronicle" packages/persistence/src/chronicle-store.ts
    rg "createChronicleContentHash" packages/persistence/src/chronicle-store.ts
  </verify>
  <acceptance_criteria>
    - `packages/persistence/src/chronicle-store.ts` contains `export interface ChronicleStore`
    - `packages/persistence/src/chronicle-store.ts` contains `createPostgresChronicleStore`

    - `pnpm --filter @cowards/persistence test -- chronicle-store.test.ts` exits 0

</task>
