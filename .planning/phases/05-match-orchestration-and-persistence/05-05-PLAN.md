# Plan 05-05: MatchSet Scoring, Status Aggregation, and End-to-End Smoke Path

<objective>
Aggregate MatchSet status and deterministic scoring from persisted Match outcomes, expose degraded/incomplete scoring when system failures occur, and provide a local smoke path that seeds, creates, executes, stores, and scores a MatchSet.
</objective>

<requirements_addressed>
MATCH-02, MATCH-04, MATCH-06, MATCH-07, DATA-03, DATA-04, TEST-05
</requirements_addressed>

<threat_model>
| Threat | Severity | Mitigation |
|--------|----------|------------|
| T-05-14: MatchSet ranking changes across runs with identical inputs | high | Sort by exact deterministic tuple: wins, surviving Soldiers, survival time/turn count, then stable revision id. |
| T-05-15: A system-failed Match is hidden inside aggregate scoring | high | Mark MatchSet scoring `degraded: true` and include `failedSystemMatches` count. |
| T-05-16: Phase 5 appears complete without proving the full persistence loop | medium | Add a dev smoke helper/test that exercises seed -> MatchSet -> worker -> Chronicle -> scoring. |
</threat_model>

<task id="05-05-01" type="test">
  <title>Implement deterministic MatchSet scoring</title>
  <read_first>
    - packages/persistence/src/schema.ts
    - packages/persistence/src/matchset-service.ts
    - packages/spec/src/types.ts
    - .planning/phases/05-match-orchestration-and-persistence/05-CONTEXT.md

  <action>
    Add `packages/persistence/src/scoring.ts` exporting:
    - `export interface MatchScoreInput { matchId: MatchId; bottomStrategyRevisionId: StrategyRevisionId; topStrategyRevisionId: StrategyRevisionId; winnerStrategyRevisionId?: StrategyRevisionId | undefined; status: MatchStatus; survivingSoldiers: number; survivalTurns: number }`
    - `export interface MatchSetStrategyScore { strategyRevisionId: StrategyRevisionId; wins: number; losses: number; draws: number; failedSystemMatches: number; survivingSoldiers: number; survivalTurns: number }`
    - `export interface MatchSetScore { degraded: boolean; complete: boolean; rankings: MatchSetStrategyScore[] }`
    - `export const scoreMatchSet = (matches: MatchScoreInput[]): MatchSetScore`

    `scoreMatchSet` must:
    - count wins/losses/draws per StrategyRevision
    - add `survivingSoldiers` and `survivalTurns`
    - set `degraded: true` when any match status is `'failed_system'`
    - set `complete: false` when any match status is not `'complete'`
    - sort rankings by wins descending, survivingSoldiers descending, survivalTurns descending, then `strategyRevisionId` ascending

    Add `packages/persistence/src/scoring.test.ts` proving exact ordering by:
    - wins first
    - cumulative surviving Soldiers second
    - cumulative survival time third
    - revision id final tie-breaker
    - `failed_system` produces `degraded: true`
  </action>
  <verify>

    rg "scoreMatchSet" packages/persistence/src/scoring.ts
    rg "survivalTurns" packages/persistence/src/scoring.ts
    rg "failed_system" packages/persistence/src/scoring.test.ts
  </verify>
  <acceptance_criteria>

</task>

  <action>
    Add `packages/persistence/src/matchset-status.ts` exporting:
    - `export const refreshMatchSetStatus = async (pool: Pool, matchSetId: MatchSetId): Promise<{ status: MatchSetStatus; scoring: MatchSetScore }>`
    - `export const listMatchStatusesForSet = async (pool: Pool, matchSetId: MatchSetId): Promise<Array<{ matchId: MatchId; status: MatchStatus }>>`

    `refreshMatchSetStatus` must:
    - load constituent Matches from `match_set_matches` ordered by `matrix_index`
    - call `scoreMatchSet`
    - set status `'pending'` when all Matches are pending
    - set status `'running'` when any Match is running and none are failed_system
    - set status `'complete'` when all Matches are complete and scoring is not degraded
    - set status `'degraded'` when all non-system-failed Matches are done and at least one Match is failed_system
    - set `match_sets.degraded` from scoring
    - persist scoring JSONB to `match_sets.scoring`

    Export from `packages/persistence/src/index.ts`.
  </action>
