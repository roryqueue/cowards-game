# Phase 165 Plan

## Goal

Build the cross-runtime abuse taxonomy and public-safe evidence format used by v1.24 probes.

## Scope

In scope:
- Define abuse probe classes by behavior first and runtime lane second.
- Define evidence records with runtime lane, probe class, expected outcome, observed outcome, failure class, privacy class, and promotion impact.
- Distinguish Strategy failures, system failures, unsupported lanes, and unavailable lanes.
- Keep evidence rerunnable locally and public-safe by default.

Out of scope:
- Running the full JS/TS, Python, Rust, or Zig probe suites.
- Promoting any runtime lane or changing Match execution.

## Tasks

1. Inventory existing sandbox, WASM/WASI, and v1.23 beta evidence schemas.
2. Define canonical probe classes for capability, resource, malformed ABI, schema/action, artifact, unavailable-lane, no-fallback, and privacy probes.
3. Define a shared evidence schema and Markdown rendering policy.
4. Add local command guidance for rerunning evidence without production credentials.
5. Validate that unsupported/unavailable lanes fail loud as non-proof.

## Success Criteria

- Taxonomy spans JS/TS, Python, Rust, Zig, and WASM/WASI lanes.
- Evidence schema captures expected/observed outcomes, failure class, privacy class, and promotion impact.
- Strategy failure and system failure are distinct.
- Unsupported/unavailable lanes are represented as fail-loud non-proof.
- Public-safe Markdown excludes source, memory, objectives, raw diagnostics, host paths, env values, tokens, DB details, package paths, and private runtime internals.
