# Plan 08-01: Canonical Engine-Generated Replay Scenario Corpus

## Objective

Create the canonical replay fixture corpus in `@cowards/test-utils` so developers can generate legal demo Chronicles for push, fall, contraction, legal Backstab, runtime failure, endgame, and a generated compound tour Match.

Purpose: Phase 8 replaces brittle hand-authored replay DTOs with inspectable engine-generated scenarios per D-01, D-02, and D-03.

Output: Exported scenario builders and manifest tests that expose expected event sequences and visual checkpoints before any web replay rendering is involved.

    - `legal-backstab` produces a Chronicle containing `BACKSTAB_RESOLVED` for a directly-behind attacker/victim pair.
    - `runtime-failure` produces a Chronicle containing `RUNTIME_VIOLATION` from a deterministic runtime violation result, not Strategy source execution.
    - `endgame` produces a terminal Chronicle containing exactly one `MATCH_ENDED`.
    - `compound-tour` is generated through the same engine/replay path and is documented as smoke coverage, not a hand-authored visual story, per D-02.
  </behavior>

    If a scenario needs multiple Rounds or Phases to reach the mechanic, encode that through deterministic activation selection and Soldier brain action plans. Keep `maxPhases` tight enough that tests fail quickly when an action script stops producing the target event.
