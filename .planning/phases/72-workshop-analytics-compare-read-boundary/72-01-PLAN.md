# Phase 72 Plan 01: Move Workshop Analytics Compare GET Behind Service

## Objective

Make `GET /api/workshop/analytics/profiles/{profileId}/compare` use a spec/service-owned comparison DTO while preserving the legacy route response and local availability behavior.

## Tasks

1. Add analytics comparison payload and service-envelope schemas to `@cowards/spec`.
2. Add `compareWorkshopAnalyticsRuns(profileId)` to `CowardsService`.
3. Reuse or extend the Workshop read web boundary helper.
4. Update the compare route to call the service boundary after the route-level availability gate.
5. Preserve storage-unavailable and null-to-404 behavior.
6. Add focused service and route tests.

## Acceptance Criteria

- External comparison response keeps `profileId`, `baseRunId`, `compareRunId`, `compatibilityEquivalent`, and `delta`.
- Not enough runs, missing profile, and compatibility mismatch continue returning the existing 404 body.
- Storage unavailable remains the current public-safe 503.
- Analytics rerun/profile save/export/gauntlet/test launch/worker/write behavior is untouched.
