# Plan 01-04: Full Local Dev Topology and Documentation

<objective>
Make the Phase 1 scaffold runnable and understandable through Docker Compose, environment examples, root documentation, and final verification.
</objective>

<requirements>
- FOUND-02: Developer can run a single command that starts the web app, worker, database, queue dependency, and local sandbox dependencies.
- TEST-07: CI or local verification command runs typecheck, lint, unit tests, integration tests, and selected E2E tests.
</requirements>

<must_haves>
- `pnpm dev` remains lightweight.
- `pnpm dev:full` starts PostgreSQL, Redis, skeletal web, and skeletal worker topology.
- Docker Compose uses real PostgreSQL and Redis.
- README documents local-only verification and explicitly says hosted CI is not included in Phase 1.
</must_haves>

<truths>
- D-09: Provide local verification only in Phase 1; do not create GitHub Actions or hosted CI yet.
- D-10: `pnpm verify` is the primary local quality gate.
- D-11: Provide two local development commands: lightweight `pnpm dev` and full `pnpm dev:full`.
- D-12: `pnpm dev` supports a fast loop without forcing all infrastructure every time.
- D-13: `pnpm dev:full` starts the planned full local topology.
- D-14: Docker Compose starts real PostgreSQL and Redis in Phase 1.
- D-15: Create minimal runnable `apps/web` and `apps/worker` in Phase 1.
</truths>

<threat_model>
| ID | Threat | Severity | Mitigation |
|----|--------|----------|------------|
| T-1-07 | Developers confuse placeholder worker/runtime topology with a safe strategy sandbox. | medium | README states strategy execution is not implemented and runtime placeholders are inert. |
| T-1-08 | Local database credentials leak or become real secrets. | low | Use `.env.example` only with local development values and keep `.env` ignored. |
</threat_model>

    Create `compose.yaml` with services:
    - `postgres` using image `postgres:18`, container name `cowards-postgres`, port `5432:5432`, environment `POSTGRES_USER=cowards`, `POSTGRES_PASSWORD=cowards`, `POSTGRES_DB=cowards_game`, volume `postgres-data:/var/lib/postgresql/data`, and healthcheck `pg_isready -U cowards -d cowards_game`.

    - `DATABASE_URL=postgresql://cowards:cowards@localhost:5432/cowards_game`

    - `NODE_ENV=development`
    Ensure `.gitignore` contains `.env`.
