# Plan 01-01: Root Workspace and Local Verification

<objective>
Create the root pnpm/Turborepo workspace and local verification gate for Coward's Game.
</objective>

<requirements>
- FOUND-01: Developer can install dependencies and run the monorepo with pnpm.
- FOUND-03: Developer can run unit, integration, and E2E test commands from the repo root.
- TEST-07: CI or local verification command runs typecheck, lint, unit tests, integration tests, and selected E2E tests.
</requirements>

<must_haves>
- The root workspace uses pnpm and Turborepo.
- Hosted CI is not added in Phase 1.
- `pnpm verify` is the local quality gate.
- Root scripts exist for `dev`, `dev:full`, `build`, `lint`, `typecheck`, `test`, and `verify`.
- The plan must not create gameplay, runtime sandbox, persistence schema, or Chronicle behavior.
</must_haves>

<truths>
- D-06: Enforce strict scaffold rules immediately in Phase 1.
- D-07: Use TypeScript project references for the build graph.
- D-08: Use ESLint/import boundary rules for explicit architecture constraints.
- D-09: Provide local verification only in Phase 1; do not create GitHub Actions or hosted CI yet.
- D-10: `pnpm verify` is the primary local quality gate.
</truths>

<threat_model>
| ID | Threat | Severity | Mitigation |
|----|--------|----------|------------|
| T-1-01 | Later phases accidentally bypass local verification because no single root gate exists. | medium | Add root `pnpm verify` script that chains lint, typecheck, and tests. |
| T-1-02 | Hosted CI is added despite user decision to keep Phase 1 local-only. | low | Do not create `.github/workflows/*`; acceptance criteria checks absence of workflow files. |
</threat_model>

    - `"dev:full": "docker compose up -d postgres redis && turbo dev --filter='./apps/*' --filter='./packages/*'"`

    Create `turbo.json` with tasks `build`, `lint`, `typecheck`, `test`, and `dev`; mark `dev` as `"cache": false` and `"persistent": true`.
    Create `.gitignore` containing `node_modules`, `.turbo`, `.next`, `dist`, `coverage`, `.env`, `.env.local`, and Docker volume folders if created.
  </action>
  <verify>

<task id="1-01-02" type="execute">

    - tsconfig.json if it exists
    - eslint.config.mjs if it exists

  <files>
    - tsconfig.base.json
    - tsconfig.json
    - eslint.config.mjs
    - prettier.config.mjs
    - vitest.config.ts
  </files>
  <action>
    Create `tsconfig.base.json` with strict TypeScript settings: `target` `ES2022`, `module` `NodeNext`, `moduleResolution` `NodeNext`, `strict` true, `noUncheckedIndexedAccess` true, `exactOptionalPropertyTypes` true, `skipLibCheck` true, and path alias `@cowards/spec` to `packages/spec/src/index.ts`.
    Create root `tsconfig.json` with `"files": []` and references to `apps/web`, `apps/worker`, `packages/spec`, `packages/engine`, `packages/runtime-js`, `packages/replay`, `packages/map-configs`, and `packages/test-utils`.
    Create `eslint.config.mjs` using flat config, `@eslint/js`, TypeScript parser/plugin if installed, and `eslint-plugin-boundaries`.
    Configure boundary rules with at least these zones:
    - `packages/spec/src/**` cannot import from `apps/**` or `packages/{engine,runtime-js,replay,map-configs,test-utils}/**`.
    - `apps/web/**` cannot import from `packages/runtime-js/**`.
    - `packages/engine/**` cannot import from `apps/**` or `packages/runtime-js/**`.
    Create `prettier.config.mjs` with semicolons disabled, single quotes false, and trailing commas `all`.
    Create `vitest.config.ts` with include pattern `["packages/**/*.test.ts", "apps/**/*.test.ts"]`.
  </action>
  <verify>
    grep -q '"strict": true' tsconfig.base.json
    grep -q 'noUncheckedIndexedAccess' tsconfig.base.json
    grep -q '@cowards/spec' tsconfig.base.json
    grep -q 'eslint-plugin-boundaries' eslint.config.mjs
    grep -q 'packages/spec/src' eslint.config.mjs
    grep -q 'apps/web' eslint.config.mjs
    grep -q 'packages/runtime-js' eslint.config.mjs
    grep -q 'packages/**/*.test.ts' vitest.config.ts
  </verify>
  <acceptance_criteria>
    - `tsconfig.base.json` contains `"moduleResolution": "NodeNext"`.
    - `tsconfig.base.json` contains `"exactOptionalPropertyTypes": true`.
    - `tsconfig.json` contains references to `apps/web`, `apps/worker`, and `packages/spec`.
    - `eslint.config.mjs` contains `eslint-plugin-boundaries`.
    - `eslint.config.mjs` contains `packages/spec/src`, `apps/web`, and `packages/runtime-js`.
    - `vitest.config.ts` contains `packages/**/*.test.ts`.
  </acceptance_criteria>
</task>
