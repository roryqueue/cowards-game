# Phase 54-01 Plan — Non-JS Strategy Product Semantics

## Scope

Deliver canonical, reusable product semantics for Strategy runtimes and apply them to validation, counted eligibility, and compact web surfaces.

## Tasks

1. [x] Extend `@cowards/spec` runtime semantics.
   - Add stable runtime product validation codes/messages.
   - Add helpers to describe runtime metadata, evaluate counted eligibility, and emit public-safe semantic issues.
   - Add tests proving JS/TS are counted, Python is experimental/non-counted, package metadata fails closed, and required validation codes exist.

2. [x] Thread semantics into validation and persistence.
   - Let `validateStrategySource` accept optional runtime metadata and include runtime semantic issues when supplied.
   - Replace duplicated competition/ladder adapter checks with the spec counted-eligibility helper and stable public messages.
   - Add focused tests for Python/non-counted runtime rejection.

3. [x] Surface compact runtime semantics.
   - Include runtime semantics in account revision summaries.
   - Show language/readiness/counted labels on account and public Strategy card pages.
   - Disable non-counted revisions in exhibition creation with the public eligibility reason.

4. [x] Verify and review.
   - Run spec, runtime-js, persistence, and web-focused tests plus typecheck.
   - Write validation, review, and summary artifacts.

## Acceptance Criteria

- NJS-01 through NJS-06 are satisfied without changing JS/TS defaults.
- Python remains experimental and non-counted.
- Counted exhibition and ladder entry reject experimental/non-counted runtime metadata with stable public-safe reasons.
- UI surfaces expose runtime semantics without adding a language picker.
