# Phase 169 Plan

## Goal

Spike direct exports as a future ABI option without changing Match execution.

## Scope

In scope:
- Direct-export proof or fail-loud non-promotion artifact.
- Memory allocation/free rules, input buffer ownership, output buffer ownership, and string/buffer encoding.
- Timeout/fuel, memory, result size, malformed output, invalid schema/action, trap, and privacy behavior.
- Rust and Zig parity attempt where feasible.
- Compatibility and rollback impact against Preview 1 stdin/stdout JSON artifact metadata.

Out of scope:
- Making direct exports eligible for Match execution.
- Replacing the active WASM/WASI Preview 1 JSON ABI.

## Tasks

1. Inspect v1.23 ABI spike evidence and current WASM/WASI validation.
2. Define the minimal direct-export ABI contract and memory ownership expectations.
3. Implement or document a local proof attempt for Rust and Zig where feasible.
4. Record fail-loud non-parity if either language/tooling path cannot satisfy the contract.
5. Generate direct-export proof and non-promotion artifacts.
6. Add guard evidence that direct-export artifacts are ineligible for Match execution.
