# Plan 12-02: Docker and No-Docker Local Parity

## Objective

Make Docker and no-Docker local development paths intentionally parallel and documented.

## Tasks

1. Update `dev:full`/Docker scripts to wait for Compose service health and run migrations/preflight before launching host-run web and worker.
2. Keep `dev:local -- --setup-only` as the no-Docker database setup path, and add an explicit local preflight command after setup.
3. Update `README.md` and `.env.example` with Docker, no-Docker, service E2E, visual E2E, and troubleshooting commands.

## Verification

- `pnpm dev:local -- --setup-only`
- `pnpm preflight:local -- --skip-redis --skip-web`
- `pnpm preflight:docker -- --skip-web`

## Done

A developer can choose Docker services or local Postgres and run the same preflight vocabulary for both.
