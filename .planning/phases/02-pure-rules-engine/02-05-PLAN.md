# Plan 02-05: Full Match Runner, Golden Tests, and Purity Gate

<objective>
Complete the Phase 2 vertical engine path by exposing `runMatch`, adding golden deterministic full-match tests, enforcing engine purity, and documenting the engine verification commands.
</objective>

<requirements_addressed>
ENG-01, ENG-02, ENG-03, ENG-04, ENG-05, ENG-06, ENG-07, ENG-08, ENG-09, ENG-10, ENG-11, ENG-12, ENG-13, ENG-14, ENG-15, ENG-16, ENG-17, ENG-18, ENG-19, ENG-20, ENG-21, TEST-01, TEST-02
</requirements_addressed>

<threat_model>
| Threat | Severity | Mitigation |
|--------|----------|------------|
| T-02-07: Full match loop never terminates | high | `runMatch` uses explicit max phase/activation guard for tests and returns failed outcome only for guard exhaustion. |
| T-02-08: Golden tests become brittle snapshots | medium | Assert deterministic equality for outcomes/event summaries and targeted state facts, not raw whole-state snapshots as the primary golden path. |
| T-02-09: Side-effect APIs creep into engine after tests pass | high | Add `purity.test.ts` and package verification script that scans engine source for forbidden APIs/imports. |
</threat_model>

<task id="02-05-01" type="execute">
  <title>Implement public runMatch API</title>
  <read_first>
    - packages/engine/src/match.ts
    - packages/engine/src/state.ts
    - packages/engine/src/activation.ts
    - packages/engine/src/contraction.ts
    - packages/engine/src/types.ts
  </read_first>
  <action>
    Add `runMatch(input)` to `packages/engine/src/match.ts`.

    `runMatch` must:
    - create initial state with `createInitialGameState`
    - accept a synchronous `StrategyRuntime`
    - advance Rounds 1-4 and Contraction until outcome exists
    - aggregate lightweight `TransitionEventSummary[]`
    - use deterministic ordering only
    - include a configurable `maxPhases` guard defaulting to a value sufficient for normal tests, such as `100`
    - return `{ state, events }`

    Export `runMatch` from `packages/engine/src/index.ts`.
  </action>
  <verify>
    pnpm --filter @cowards/engine test -- match.test.ts
    rg "export const runMatch" packages/engine/src/match.ts
    rg "runMatch" packages/engine/src/index.ts
  </verify>
  <acceptance_criteria>
    - `packages/engine/src/match.ts` contains `export const runMatch`
    - `packages/engine/src/match.ts` contains `maxPhases`
    - `packages/engine/src/index.ts` exports from `./match.js`
    - `pnpm --filter @cowards/engine test -- match.test.ts` exits 0
  </acceptance_criteria>
</task>
