# Plan 02-03: Actions, Movement, Collision, Push, and Backstab

<objective>
Implement the canonical action and movement resolution pipeline, including TURN, TURN_TO_STONE, MOVE, reversal, blocked movement, head-to-head collision, side push, clarified Backstab boundary rules, and no-advance cleanup.
</objective>

<requirements_addressed>
ENG-07, ENG-08, ENG-09, ENG-10, ENG-11, ENG-12, ENG-13, ENG-14, ENG-15, ENG-16, ENG-21, TEST-01
</requirements_addressed>

<threat_model>
| Threat | Severity | Mitigation |
|--------|----------|------------|
| T-02-04: Movement ordering produces exploitable Backstab escapes | high | Apply activation-start and activation-end all-soldier Backstab boundary checks using simultaneous snapshots. |
| T-02-05: Collision or push mutates reversal history incorrectly | medium | Test that pushes do not update `lastSuccessfulMoveDirection`; only successful voluntary MOVE updates it. |
| T-02-06: Multiple terminal reasons make outcomes nondeterministic | medium | Return one terminal reason per activation while allowing bundled effects inside a successful advance. |
</threat_model>
