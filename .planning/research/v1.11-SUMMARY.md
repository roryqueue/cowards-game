# Research Summary: v1.11 Remaining Web Read Boundary Burn-Down and Live Go Readiness Evidence

**Project:** Coward's Game
**Milestone:** v1.11 Remaining Web Read Boundary Burn-Down and Live Go Readiness Evidence
**Date:** 2026-05-23
**Confidence:** HIGH for Workshop test-summary and analytics-compare read migration; HIGH for live Go evidence-only validation; MEDIUM for exact report-only count reduction until implementation proves dependency cleanup.

## Executive Recommendation

v1.11 should continue service-boundary debt reduction without chasing the remaining broad web report-only offenses indiscriminately. The current baseline is:

```text
strict_offenses=0 report_only_offenses=30
```

Most remaining offenses are writes, source-bearing owner reads, replay owner-debug/private Chronicle assembly, or mixed Workshop source/runtime/export surfaces. The safest next user-facing read slice is:

1. Workshop test-summary GET read.
2. Workshop analytics-compare GET read.
3. Source-free type cleanup only where the selected DTO ownership is proven.

Live Go readiness is mature enough to require as validation evidence, but not mature enough for production promotion. v1.11 should require `pnpm go:parity`, `pnpm boundary:monitors`, and required live Go topology evidence while keeping production web traffic on the TypeScript service path.

## Current Boundary Evidence

- `pnpm boundary:imports` passes with `strict_offenses=0 report_only_offenses=30`.
- The remaining report-only offenses cluster around account forks/source, auth mutations, exhibition creation, ladder mutations/scheduling, governance mutations, the legacy `competitive/server` facade, replay owner-debug/private Chronicle assembly, Workshop source/test/export/rerun flows, and Workshop broad type imports.
- `@cowards/service` already owns public MatchSet summary, public replay metadata, public Strategy page, public player page, public ladder page, auth session, account revision list, analytics run summary, and Workshop analytics snapshot reads.
- Go currently serves five fixture-backed GET routes: health, public MatchSet summary, public replay metadata, public Strategy page, and owner-scoped analytics run summary.

## Selected v1.11 Scope

### 1. Boundary Debt Rebaseline

Capture a v1.11 ownership matrix before implementation. Classify all 30 offenses into:

- selected Workshop read candidates,
- intentionally deferred writes/mutations/orchestration,
- Strategy source/private owner reads,
- replay owner-debug/private Chronicle surfaces,
- Workshop source/runtime/export flows,
- cleanup-only type/test debt.

This prevents cleanup-by-hiding through broad facades.

### 2. Workshop Test Summary Read DTO

Move `GET /api/workshop/tests/{matchSetId}` behind a spec/service-owned read DTO. Preserve current user-visible fields:

- MatchSet id,
- status,
- match count,
- Match ids,
- match summaries,
- scoring,
- replay availability.

Do not move test launch, source validation, source save, source retrieval, worker execution, Match orchestration, or runtime behavior.

### 3. Workshop Analytics Compare Read DTO

Move `GET /api/workshop/analytics/profiles/{profileId}/compare` behind a spec/service-owned comparison DTO. Preserve current comparison behavior:

- profile id,
- base and comparison run ids,
- compatibility/equivalence,
- W-L-D,
- points,
- evidence counts,
- deltas.

Do not move analytics rerun, profile save, export, gauntlet execution, test Match launch, worker execution, or persistence writes.

### 4. Boundary Enforcement and Type Cleanup

Promote only proven migrated route files and safe dependencies to strict import enforcement. Source-free type cleanup is allowed only when it removes real broad web report-only fingerprints without promoting source-bearing Workshop templates, samples, private source payloads, or runtime internals into public/service outputs.

The final import boundary target is:

```text
strict_offenses=0 report_only_offenses<30
```

### 5. Live Go Evidence-Only Validation

Require live Go evidence in the milestone gate:

- `pnpm go:parity`
- `pnpm boundary:monitors`
- local Go backend started with fixture owner-token configuration
- `pnpm topology:check -- --require-go --json`

Required live Go checks should include health, public MatchSet summary, public replay metadata, public Strategy page, unauthenticated owner analytics rejection, route privacy, and diagnostic privacy.

## Explicit Non-Goals

- Go writes, auth/session mutation, ladder writes, Match orchestration, jobs, migrations, persistence writes, Strategy source retrieval, or Strategy execution.
- Production web routing to Go.
- Silent TypeScript fallback during required live Go validation.
- Production hostile-code sandbox promotion.
- Counted Python or other non-JS MatchSets, ladders, gauntlets, analytics comparisons, or standings.
- Workshop Strategy source, source save, validation/test execution, test launch, analytics rerun, profile save, or export migration.
- Full replay projection, owner-debug replay migration, private Chronicle assembly, or replay fixture rewrite.
- Rule, Chronicle, scoring, terminology, engine, or deterministic runtime semantics changes.

Expected outcomes:

- `strict_offenses=0`.
- `report_only_offenses` drops below 30 through real fingerprint removal.
- Public/service/Go/topology/monitor outputs omit Strategy source, StrategyMemory, SoldierMemory, objective payloads, owner debug, raw Awareness Grid, stack traces, stderr, sessions, tokens, host paths, and private runtime internals by default.
- Live Go evidence is required for validation but does not route production web traffic to Go.
- Runtime isolation and non-JS remain evidence-only/non-counted.

## Key Risks

- Migrating Workshop reads through a new broad facade without reducing direct persistence debt.
- Accidentally moving Workshop launch, source, rerun, export, or runtime behavior under the read DTO work.
- Treating replay as a low-risk read surface even though the remaining replay imports touch private Chronicle and owner-debug assembly.
- Treating live Go evidence as production promotion.
- Letting required Go validation silently fall back to TypeScript when Go is unavailable or divergent.
