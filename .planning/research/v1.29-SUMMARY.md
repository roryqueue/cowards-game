**Project:** Coward's Game
**Milestone:** v1.29 Replay and Result Trust Polish

## Executive Summary

v1.29 should live entirely on the app/public UX side of the existing Match execution contract. v1.28 deliberately kept public result/replay contract expansion out of scope while adding internal operations recovery and proof artifacts. That leaves a good next milestone: improve how public pages explain existing completed, queued, running, degraded, failed, stale-artifact, unavailable-runtime, malformed-runtime-result, missing-Chronicle, and no-result states without touching Go execution semantics, runtime-service behavior, retry/recovery policy, quarantine, scoring, Chronicle persistence, or public DTO shape.

The main implementation surface is the web result/replay presentation layer and its tests/proof scripts:

- `apps/web/app/matchsets/[matchSetId]/page.tsx`
- `apps/web/app/matchsets/evidence-copy.ts`
- `apps/web/app/matches/[matchId]/replay/replay-client.tsx`
- `apps/web/app/matches/[matchId]/replay/replay-unavailable.tsx`

## Baseline Findings

- v1.28 is complete and tagged; `match-execution-app-v1` remains frozen.
- Current fixture coverage includes complete, running, queued, strategy failure, system failure, timeout, unavailable runtime, malformed runtime result, stale artifact, and public-safe replay.
- `missing_chronicle` and `no_result` are already contract vocabulary, but they have weaker public page proof than the core fixture scenarios.
- Result copy already explains broad status, lifecycle, availability, retry policy, runtime evidence, candidate lane limits, proof limits, entrants, and privacy.
- Replay pages already show public evidence rows and board realism validation, while unavailable replay pages are much thinner.
- Board realism checks already reject invalid bounds, out-of-bounds visible Soldiers/STONE terrain, FALLEN visible pieces, overlap, and canonical starting position drift.
- Boundary monitors already validate v1.25/v1.26/v1.28 contract compatibility, privacy, ownership, runtime eligibility, and non-claims.

## Recommended Scope

- Keep `match-execution-app-v1` frozen with no new public DTO fields.
- Improve public copy and layout using existing DTO fields and existing public-safe projection data.
- Add or extend fixture-backed evidence only inside the existing test/dev fixture adapter.
- Expand browser proof around result and replay states, including missing Chronicle and no-result public pages.
- Add privacy scan coverage for copy, fixtures, public pages, and proof artifacts.
- Add a v1.29 proof artifact and boundary monitor that explicitly proves no contract drift.

## Watch Outs

- Do not represent quarantine, operator action IDs, requeue/rerun details, or internal recovery payloads as public UX state.
- Do not add public DTO fields to solve copy or layout problems.
- Do not change Go match execution, runtime-service behavior, retry/recovery policy, quarantine semantics, job lifecycle, MatchSet scoring, Chronicle persistence, or operator controls.
- Do not expose Strategy source, StrategyMemory, SoldierMemory, objective payloads, raw diagnostics, host paths, env values, tokens, DB details, package paths, runtime internals, quarantine details, operator details, or recovery payloads.
- Do not use v1.27 as a baseline unless it has been explicitly merged into this checkout.
