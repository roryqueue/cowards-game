# Architecture Research: v1.12 Go Backend Promotion Readiness and Cutover Plan

**Project:** Coward's Game
**Researched:** 2026-05-23
**Mode:** Architecture research
**Confidence:** HIGH for current boundaries and generated parity fixture flow; MEDIUM for production switch mechanics until implemented and browser/topology verified.

## Executive Summary

v1.12 should add a promotion-readiness architecture, not a backend rewrite. The existing TypeScript stack already has the right ownership shape: `@cowards/spec` owns route contracts and DTO privacy rules, `@cowards/service` owns service DTO construction, Next route handlers call thin web boundary modules, and `apps/go-backend` serves read-only TypeScript-generated fixture DTOs behind a GET-only allowlist. The missing piece is an explicit, route-scoped cutover switch that can prove one production web read can use Go without silent fallback, private data leakage, or ownership creep.

The recommended v1.12 promotion candidate is `GET /public/strategies/{strategyId}` as consumed by `apps/web/app/strategies/[strategyId]/page.tsx`, not owner analytics, replay payloads, Strategy source, Match orchestration, or any mutation. Public Strategy page is already public, source-free, present in `SERVICE_API_ROUTES`, generated by `scripts/generate-go-parity-fixtures.ts`, served by Go, covered by Go topology, and strict-import gated on the web side through `apps/web/lib/public-service-boundary.ts`.

The switch must live below web routes and above concrete service clients. Web routes and pages should keep calling the same boundary methods. `apps/web/lib/public-service-adapter.ts` should become the route-selection point for approved public reads, choosing either the local TypeScript `createCowardsLocalService` implementation or a new typed Go HTTP client. The switch should be allowlisted by service route id, explicit per route, default to TypeScript, fail closed when Go is selected but unavailable/divergent, and log sanitized route-level decisions.

Production rollback should be a one-line switch back to TypeScript plus logs/topology evidence proving no silent fallback occurred. v1.12 may promote at most one narrow read route only after parity, topology, import-boundary, privacy, switch, rollback, and log checks all pass. Go writes, auth/session mutation, jobs, migrations, persistence ownership, Strategy source retrieval, Strategy execution, and full replay/owner-debug projection remain out of scope.

## Current Architecture To Preserve

```txt
Next route/page
  -> route-specific web boundary module
  -> public/account/workshop service adapter
  -> @cowards/service CowardsService
  -> @cowards/spec schemas + privacy guards
  -> @cowards/persistence read/write helpers as allowed

Generated parity path:
@cowards/service canonical DTO
  -> @cowards/spec schema + privacy validation
  -> scripts/generate-go-parity-fixtures.ts
  -> apps/go-backend/testdata/service-fixtures/*.json
  -> apps/go-backend read-only handlers
  -> pnpm go:parity, pnpm topology:check, pnpm boundary:monitors
```

| Boundary | Current owner | v1.12 rule |
| --- | --- | --- |
| Route contracts and DTO schemas | `packages/spec/src/service.ts`, `packages/spec/src/schemas.ts` | Source of truth for route ids, method/path, auth scope, privacy class, response schemas, examples, and generated artifacts. |
| TypeScript service implementation | `packages/service/src/index.ts` | Canonical DTO producer and validation path. Go fixtures continue to be generated from this service. |
| Web public read boundary | `apps/web/lib/public-service-boundary.ts` | Stable caller API for pages/routes. Do not put Go URLs or fallback logic in route handlers. |
| Web concrete service adapter | `apps/web/lib/public-service-adapter.ts` | Correct place to add route-scoped TypeScript-vs-Go selection for approved public reads. |
| Workshop/account boundaries | `apps/web/lib/workshop-read-service-boundary.ts`, `apps/web/lib/account-service-boundary.ts` | Stay TypeScript-owned in v1.12; do not route owner/session/workshop reads to Go. |
| Go backend | `apps/go-backend/main.go` | Read-only allowlisted HTTP server serving generated fixtures. No DB ownership, writes, jobs, migrations, or Strategy execution. |
| Parity generation | `scripts/generate-go-parity-fixtures.ts` | Generates Go fixture JSON, route manifest, and checksum source from TypeScript service outputs. |
| Topology checks | `scripts/check-local-topology.ts` | Required-live Go evidence must fail loudly when Go is absent or divergent. |
| Boundary monitors | `scripts/check-boundary-monitors.ts`, `scripts/check-service-boundary-imports.ts` | Drift gate for OpenAPI, privacy, Go route manifest, known web import debt, runtime guardrails, and topology. |

## Recommended Promotion Candidate

Promote only `getPublicStrategyPage` if any route is promoted.

| Criterion | Assessment |
| --- | --- |
| Public-safe DTO | `PublicStrategyPageServiceDtoSchema` contains public card metadata: ids, name, tags, author handle, source hash/bytes, runtime metadata, compatibility, validation status, records, result/replay links. It excludes Strategy source and memories. |
| Canonical route contract | `SERVICE_API_ROUTES.getPublicStrategyPage` defines `GET /public/strategies/{strategyId}` with `authScope: public` and `privacyClass: public`. |
| TypeScript service support | `CowardsService.getPublicStrategyPage` builds DTOs through `buildPublicStrategyCardDto`, parses schema, and runs `assertPublicServiceDtoLeakSafe`. |
| Go support | `apps/go-backend` route inventory serves `GET /public/strategies/{strategyId}` from `public-strategy-page.json`. |
| Existing topology | `pnpm topology:check -- --require-go --json` already smokes public Strategy page live against Go. |
| Web boundary | `apps/web/app/strategies/[strategyId]/page.tsx` calls `getPublicStrategyCard`, which calls `publicReadService.getPublicStrategyPage`. |
| Rollback simplicity | Switching this one route back to TypeScript only affects public Strategy page reads. No owner session, source retrieval, Match execution, or writes are involved. |

Do not promote `getAnalyticsRunSummary` in v1.12. It is owner-scoped, bearer-token based in Go fixture mode, not the current web owner auth/session model, and too easy to confuse evidence-only auth with production authorization. Do not promote replay beyond metadata; public replay projection and owner debug remain sensitive.

## Promotion-Readiness Components

### 1. Route Promotion Registry

Add a small registry in the web/service boundary layer, not in React pages:

```ts
type PromotableGoReadRouteId = "getPublicStrategyPage"

interface GoReadPromotionConfig {
  backend: "ts" | "go"
  promotedRoutes: readonly PromotableGoReadRouteId[]
  goBaseUrl: string | null
  fallback: "disabled"
}
```

Recommended environment semantics:

| Env var | Default | Meaning |
| --- | --- | --- |
| `COWARDS_PUBLIC_READ_BACKEND` | `ts` | Global read backend mode. Values: `ts`, `go`. |
| `COWARDS_GO_PROMOTED_ROUTES` | empty | Comma-separated allowlist of promoted route ids. v1.12 allows only `getPublicStrategyPage`. |
| `COWARDS_GO_BACKEND_URL` | unset | Required when any route is promoted to Go. |
| `COWARDS_GO_READ_FALLBACK` | `disabled` | Must remain disabled. Do not support silent TypeScript fallback in v1.12. |

Fail startup or first call closed when `COWARDS_PUBLIC_READ_BACKEND=go` but the route is not allowlisted, `COWARDS_GO_BACKEND_URL` is missing, the method is not GET, or the service route privacy class is not public.

### 2. Typed Go Public Read Client

Add a typed client beside the existing local adapter. Keep it narrow:

```txt
apps/web/lib/public-service-adapter.ts
  -> local TypeScript CowardsService for default paths
  -> GoPublicReadClient for allowlisted promoted paths

GoPublicReadClient
  -> GET {COWARDS_GO_BACKEND_URL}/public/strategies/{strategyId}
  -> parse PublicStrategyPageServiceDtoSchema
  -> assertPublicServiceDtoLeakSafe
  -> map 404 to null
  -> throw public-safe upstream error for network/5xx/schema/privacy failures
```

Do not add a generic proxy or route-by-path string switch. Route by `SERVICE_API_ROUTES.getPublicStrategyPage.id`, use `SERVICE_API_ROUTES` metadata to construct/check the request, and schema-parse every response. The client should not know about persistence, cookies, sessions, Strategy source, runtime execution, or Match orchestration.

### 3. Switch Semantics

The switch is explicit and no-fallback:

```txt
backend=ts
  -> all web reads use local TypeScript CowardsService

backend=go + route not promoted
  -> fail closed for attempted Go route selection

backend=go + getPublicStrategyPage promoted + Go returns 200 valid DTO
  -> serve Go DTO

backend=go + getPublicStrategyPage promoted + Go returns 404 valid ServiceErrorDto
  -> return null and preserve existing Strategy-not-found UX

backend=go + network error / 5xx / non-JSON / schema drift / privacy violation
  -> throw upstream-unavailable style error; do not call TypeScript service
```

This is stricter than user-facing resilience, deliberately. The milestone is proving promotion readiness; a silent TypeScript fallback would hide the exact failure v1.12 is meant to expose.

### 4. Route-Level Logs

Add sanitized structured logs only at the adapter/switch layer:

| Event | Required fields | Forbidden fields |
| --- | --- | --- |
| route decision | route id, selected backend, promoted bool, fallback disabled, request id if available | Strategy source, sessions, tokens, bearer values, owner debug, host paths |
| Go success | route id, backend `go`, status, duration bucket, schema version | response body, source text, private runtime data |
| Go not found | route id, backend `go`, status 404 | raw error details beyond public-safe code/message |
| Go failure | route id, backend `go`, failure class `network`/`http`/`schema`/`privacy`, status if available | token, URL query secrets, stack traces in public output |
| rollback evidence | route id, selected backend `ts`, previous promoted route if known | secrets or private payloads |

Logs should be useful locally and in CI without introducing a production observability stack. Existing topology diagnostic redaction in `check-local-topology.ts` is the model: sanitize URLs, redact bearer tokens, and pass public DTO leak checks on diagnostics.

## Data Flow After v1.12 Switch

```txt
Strategy page request
  -> apps/web/app/strategies/[strategyId]/page.tsx
  -> getPublicStrategyCard(strategyId)
  -> publicReadService.getPublicStrategyPage(strategyId)
  -> promotion registry decides:
       ts: createCowardsLocalService(...).getPublicStrategyPage(...)
       go: GoPublicReadClient.getPublicStrategyPage(...)
  -> PublicStrategyPageServiceDtoSchema.parse(...)
  -> assertPublicServiceDtoLeakSafe(...)
  -> page renders PublicStrategyCardDto only
```

Route handlers for MatchSet summary, replay metadata, player page, ladder page, account reads, Workshop reads, and all mutations should continue using the TypeScript service path in v1.12 unless explicitly included in a later roadmap phase.

## Build Order

1. **Scope and Baseline Lock**
   - Record current `pnpm boundary:imports` count, `SERVICE_API_ROUTES` Go inventory, strict migrated files, topology behavior, and existing Go fixtures.
   - Name `getPublicStrategyPage` as the only eligible promotion candidate and document all non-goals.
   - Output should include rollback criteria before any code switch exists.

2. **Promotion Registry and Go Client**
   - Add route id allowlist and env parsing.
   - Add a narrow `GoPublicReadClient` for `getPublicStrategyPage`.
   - Parse all Go responses with `@cowards/spec` schemas and run privacy guards.
   - Keep web pages/routes unchanged except through the adapter dependency.

3. **No-Fallback Switch Integration**
   - Integrate the registry in `apps/web/lib/public-service-adapter.ts`.
   - Preserve default TypeScript behavior.
   - Add tests for TypeScript default, Go success, Go 404, Go unavailable, schema drift, privacy violation, missing URL, and unapproved route attempts.

4. **Parity and Fixture Hardening**
   - Extend `pnpm go:parity` tests if needed so public Strategy route fixture checks cover success, missing resource error, schema validation, private marker rejection, checksum drift, and route manifest drift.
   - Keep fixtures generated from TypeScript service, not hand-maintained Go payloads.

5. **Topology Cutover Evidence**
   - Add topology mode that can exercise the web Strategy page through the Go-promoted adapter, not just direct Go smoke.
   - Required evidence should prove:
     - Go live direct route passes.
     - Web route/page can read through Go when explicitly switched.
     - Web route/page fails loudly when switched to Go and Go is unavailable.
     - Web route/page returns to TypeScript when switch is reverted.

6. **Boundary, Privacy, and Log Monitors**
   - Update `pnpm boundary:monitors` so the promoted route registry cannot include non-GET, owner, mutation, Strategy source, replay owner-debug, runtime, worker, or persistence-ownership routes.
   - Add log/diagnostic privacy assertions for switch decisions and failures.
   - Ensure strict import boundaries still forbid persistence/runtime imports in promoted route callers.

7. **Optional Narrow Promotion**
   - Only after the above gates pass, set the explicit switch for `getPublicStrategyPage` in the selected environment.
   - Document rollback command/env diff and evidence.
   - Stop at one route. Do not use success here as permission for a second route in the same milestone.

## Rollback Architecture

Rollback should not require code edits:

```txt
COWARDS_PUBLIC_READ_BACKEND=ts
COWARDS_GO_PROMOTED_ROUTES=
COWARDS_GO_BACKEND_URL=   # unset in the web environment
```

Expected rollback behavior:

- `apps/web/lib/public-service-adapter.ts` returns to local `createCowardsLocalService` for `getPublicStrategyPage`.
- Go may be stopped without affecting public Strategy page reads.
- Logs show route id `getPublicStrategyPage`, selected backend `ts`, and fallback disabled.
- `pnpm topology:check` may still optionally smoke direct Go, but production web reads no longer depend on it.
- No fixtures, Go handlers, service DTOs, or web route handlers need to be reverted.

## Anti-Patterns To Avoid

### Generic Go Proxy

**What goes wrong:** A path-based proxy lets route handlers forward arbitrary reads to Go.
**Why bad:** It bypasses `SERVICE_API_ROUTES`, weakens privacy classification, and makes accidental owner/source/replay promotion likely.
**Instead:** Route by service route id and allowlist exactly `getPublicStrategyPage`.

### Silent TypeScript Fallback

**What goes wrong:** Go outage or schema drift quietly calls `@cowards/service`.
**Why bad:** v1.12 would produce false confidence and hide production cutover failures.
**Instead:** Default to TypeScript, but once a route is explicitly Go-selected, fail closed until rollback env changes are applied.

### Switch In React Or Route Handlers

**What goes wrong:** Pages/routes learn backend ownership and start carrying transport logic.
**Why bad:** It spreads cutover semantics, complicates rollback, and violates the existing route-thin architecture.
**Instead:** Keep switch logic in `apps/web/lib/public-service-adapter.ts` and a small Go client.

### Promoting Owner Or Source-Bearing Routes

**What goes wrong:** Account, Workshop source, owner analytics, replay debug, or Strategy source routes move to Go before auth/session and privacy semantics are production-equivalent.
**Why bad:** Public/private boundaries become ambiguous, and rollback can leak or orphan owner workflows.
**Instead:** Keep owner/session/source/runtime/replay-private paths TypeScript-owned.

### Treating Fixtures As Persistence Ownership

**What goes wrong:** Go fixture success is interpreted as Go owning live data reads broadly.
**Why bad:** Fixtures prove DTO parity, not production DB query semantics or transaction behavior.
**Instead:** Use fixture-backed Go only for one route switch proof; defer live Go persistence ownership to a later milestone.

## Roadmap Implications

Recommended v1.12 phases:

1. **Promotion Scope and Baseline** - Lock `getPublicStrategyPage` as the only candidate, record current parity/topology/import evidence, and document non-goals.
2. **Go Read Switch Contract** - Add route promotion registry, env semantics, typed Go client, and no-fallback behavior.
3. **Web Adapter Cutover Path** - Wire the switch under `publicReadService` while keeping page/route callers stable.
4. **Parity, Privacy, and Boundary Gates** - Harden Go fixtures, monitors, strict imports, and log privacy around the promoted route.
5. **Live Cutover and Rollback Evidence** - Prove direct Go, web-through-Go, failure-without-fallback, and rollback-to-TypeScript.
6. **Milestone Verification** - Run full contract, service, web, Go, topology, boundary, replay privacy, typecheck, and formatting gates; promote at most one route only if all pass.

## Open Questions

- Should v1.12 use `COWARDS_PUBLIC_READ_BACKEND=go` plus route allowlist, or a single `COWARDS_GO_PROMOTED_ROUTES=getPublicStrategyPage` flag? Recommendation: use both so global mode and route scope are explicit.
- Should a promoted route failure return a user-facing 503 or preserve the existing page error path? Recommendation: throw a typed upstream-unavailable error at the adapter and map it to existing safe error handling; do not leak Go response bodies.
- Should live web-through-Go evidence be Playwright browser evidence or topology fetch evidence? Recommendation: topology fetch first, browser smoke if the route renders materially different page states.
