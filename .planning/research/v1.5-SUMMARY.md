# v1.5 Research Summary: Workshop Tools and Advanced Strategy Evidence

**Project:** Coward's Game
**Milestone:** v1.5 Strategy Workshop Power Tools and Advanced Strategy Library
**Synthesized:** 2026-05-20
**Inputs:** `.planning/research/v1.5-WORKSHOP-UX.md`, `.planning/research/v1.5-STRATEGY-LIBRARY.md`

## Agreement

The Workshop UX and Strategy Library research tracks agree on the v1.5 product direction:

1. The Strategy Workshop should become a deterministic authoring lab by extending existing MatchSet, worker job, scoring, Chronicle, replay, and privacy DTO infrastructure. v1.5 should not introduce a new Strategy execution path.
2. Gauntlets should be represented as persisted, versioned matrix profiles. Result deltas are meaningful only when the compared revisions used the same profile: opponent set, preset, seed/mirror policy, scoring policy, rule/spec/Chronicle/runtime versions, and compatibility metadata.
3. Revision comparison should combine owner-authorized source diff, metadata deltas, validation/runtime diagnostic deltas, and deterministic result deltas. Public pages may expose hashes, lineage, records, and replay evidence, but not private source or memory data by default.
4. Diagnostics should be structured, grouped by category, and linked to replay evidence where available. Strategy failures must remain distinct from system failures.
5. The Advanced Strategy Library should be a distinct tier: stronger benchmark/seed Strategies that teach broad archetypes and stress player submissions. It is not a permanent ranking promise.
6. Evidence should combine starter gauntlets, advanced-vs-advanced round robin results, curated example MatchSets, Chronicle-derived behavior signals, and a completed v1.5 demo tournament.
7. Diversity is an acceptance gate, not decoration. v1.5 should target 10 advanced Strategies and accept 8-9 only when stronger quality/diversity evidence justifies the reduction.
8. Browser verification must cover the visible evidence path: Advanced Library, Strategy cards, MatchSet result pages, tournament page, player/profile pages if applicable, and representative replays.

## Balanced Direction

v1.5 should sequence work so tooling creates trustworthy evidence before the final library is accepted:

1. Build the Workshop matrix/comparison/diagnostics surface against existing v1.4 Starters first.
2. Analyze v1.4 result data and define the evidence model for archetype claims.
3. Design the Advanced Strategy library and publish it as a separate tier.
4. Run deterministic starter and advanced gauntlets, then tune only through documented review triggers.
5. Generate curated MatchSets and a completed demo tournament from the accepted advanced set.
6. Browser-verify representative public and owner-authorized flows, then update documentation before milestone completion.

## Decisions for Requirements

| Decision | Rationale |
| --- | --- |
| Reuse MatchSet/jobs/replay instead of a new runner | Preserves deterministic execution, runtime isolation, Chronicle provenance, and existing result pages. |
| Persist gauntlet profiles | Prevents misleading revision comparisons and enables reproducible local reports. |
| Add owner-only source diff | Monaco is already present, and advanced iteration needs revision-level code comparison. |
| Publish Advanced Library separately from Starter Library | Starter Strategies teach basics; Advanced Strategies act as stronger benchmarks and seed opponents. |
| Require archetype coverage | Prevents overfitting to v1.4 winners and protects pressure, defense, mobility, trap, adaptation, contraction, and memory play. |
| Prefer round-robin demo tournament | An 8-10 entrant round robin better demonstrates matchup texture than single elimination. |
| Keep tournament framing non-durable | v1.5 generates local demo evidence, not official public tournament operations or permanent ratings. |

## Implementation Guardrails

- No Strategy code may run in the web/API process, browser, or Node `vm`.
- Engine logic remains pure, deterministic, serializable, and side-effect free.
- Public outputs must not expose Strategy source, StrategyMemory, SoldierMemory, objective payloads, owner debug, raw Awareness Grid internals, private runtime internals, or stack traces by default.
- Strategy Revisions used in gauntlets, MatchSets, and tournaments must be immutable snapshots.
- Evidence summaries must include rule/spec/Chronicle/runtime compatibility labels so v1.4 evidence and v1.5-generated evidence are not confused.
- Runtime/system failures must be reported separately from Strategy failures and must not decide completed demo tournament standings.

## Requirement Themes

1. Workshop power tools: deterministic matrix, comparison, diagnostics, replay handoff, performance summaries.
2. v1.4 result analysis and evidence model: reusable metrics, behavior signals, review triggers, local report.
3. Advanced library: distinct tier, archetype taxonomy, system-owned immutable seeds, public cards, fork/apply path.
4. Validation: starter gauntlet, advanced round robin, curated counter gauntlets, replay review, tuning notes.
5. Demo evidence: example MatchSets, completed demo tournament, representative local links.
6. Verification and docs: browser verification, privacy regression tests, documentation updates, acceptance report.
