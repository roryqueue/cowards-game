# v1.33 Milestone Audit

**Milestone:** Source Artifact Provenance and TinyGo WASI Spike
**Date:** 2026-06-01
**Status:** Passed

## Summary

v1.33 adds artifact provenance to TypeScript and Python source-language Strategy paths, keeps Rust and Zig on immutable WASM/WASI artifacts, records TinyGo as a spike-only candidate, and updates product evidence so public language surfaces do not overclaim sandbox maturity or expose private artifact bytes.

## Requirements

- Requirements complete: 24/24
- Phases complete: 234-237
- Provider contract: `strategy-language-provider-contract-v1.33`
- Counted production language posture:
  - TypeScript: transpiled JavaScript source artifact provenance through `strategy-language-provider-js-ts`
  - Python: normalized source bundle provenance through `strategy-language-provider-python`
  - Rust: immutable WASM/WASI artifact provenance through `strategy-language-provider-rust-wasi`
  - Zig: immutable WASM/WASI artifact provenance through `strategy-language-provider-zig-wasi`
- Spike-only language posture:
  - TinyGo: not production-enabled and not counted eligible in v1.33

## Final Decision

TypeScript and Python source-language submissions now carry deterministic source artifacts with artifact-bound provider proof before counted play. Runtime execution validates the artifact and fails closed on stale, missing, or mismatched provenance instead of silently rebuilding source at execution time.

TinyGo remains a future candidate. After local TinyGo installation, the spike compiled a WASI artifact and executed the minimal stdin/stdout JSON ABI through Wasmtime, but the artifact imports production-forbidden WASI capabilities. The milestone records public-safe evidence and defers promotion without adding TinyGo to production language pickers or counted eligibility gates.

## Audit Findings

All findings were fixed:

- The public strategy artifact manifest was stale after source artifact schema changes. It was regenerated and rechecked.
- The web boundary drift baseline changed after the Workshop server started importing public metadata redaction. The known baseline was updated and the boundary monitor passed.
- A brittle Learn page text assertion failed after line wrapping changed. It was replaced with semantic assertions over the supported language copy.
- A monitor label still referenced the previous milestone's language-provider check name. It was updated to v1.33.
- After TinyGo was installed, the old spike harness compiled but did not audit imports or runtime behavior. It now audits imports, executes the minimal ABI through Wasmtime, records determinism/invalid-output/timeout behavior, and tolerates volatile TinyGo build hashes/timings during `--check`.
- TinyGo GA constraints were added to `.planning/research/TINYGO-GA-CONSTRAINTS.md`, and production Learn/MatchSet evidence copy was scrubbed so TinyGo appears only in planning/spike artifacts and negative app tests.

## Privacy

Public result, replay, Workshop, manifest, proof, and spike evidence does not expose Strategy source, StrategyMemory, SoldierMemory, objective payloads, `sourceArtifact.bytesBase64`, compiled artifact bytes, raw diagnostics, host paths, env values, tokens, DB details, package paths, or private runtime internals by default.

## Compatibility Notes

- The Go backend still preserves legacy TypeScript fork-manifest compatibility for built-in manifest lookups. TypeScript provider-proof enforcement is active in runtime-service, Workshop submission, persistence, and execution paths, while Go's legacy TypeScript manifest branch remains a deliberate compatibility exception until built-in manifests can be provider-signed without environment-specific HMAC coupling.
- TinyGo was not promoted because its compiled WASI artifact imports production-forbidden capabilities (`clock_time_get`, args, and random), even though the minimal ABI execution proof passed.
- Production product surfaces intentionally do not mention TinyGo until a future milestone approves GA.
