# Phase 14 Plan 01: Auth Persistence and Session Contracts

**Status:** Planned
**Requirements:** OWN-01, OWN-02, OWN-03

## Goal

Add minimal username/password account and server-side session persistence for Competitive Alpha.

## Tasks

1. Add additive migration tables/columns for credentials, unique username/handle, and sessions.
2. Implement password hashing and verification with Node crypto.
3. Implement session creation, lookup, revocation, and expiry helpers.
4. Add focused tests for signup uniqueness, password verification, session expiry, and revocation.

## Acceptance

- Password hashes are never stored as plaintext.
- Sessions are server-side, expire, and can be revoked on sign out.
- Existing Workshop seed data still works.
