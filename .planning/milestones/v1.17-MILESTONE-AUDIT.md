## Audit Findings And Fixes

| Finding | Severity | Fix |
| --- | --- | --- |
| Python validation import pulled runtime adapter code into Next page render. | High | Added `@cowards/runtime-python/validation` subpath and moved runtime metadata to side-effect-light `metadata.ts`. |
| Broker registry validation could hide raw ABI drift through coercion. | High | Added raw ABI/language/adapter/package checks before metadata coercion. |
| Runtime-service Python proof fixture returned invalid activation orders. | Medium | Added objective-bearing activation order output and zero-violation assertion. |
| Runtime-service Python proof used a too-tight 250 ms timeout under package-level Vitest concurrency. | Medium | Switched test request to default runtime timeout. |
| Web ownership tests used cwd-sensitive file paths and blocked package test verification. | Medium | Resolved test file paths from repo root. |
| Initial registry artifact did not include all spec broker entries. | Medium | Expanded artifact to all seven JS/TS/Python registry entries and monitor-checked it against spec. |
| Full boundary monitor found stale TypeScript backend inventory artifacts. | Medium | Regenerated the v1.16 TypeScript backend inventory and reran `pnpm boundary:monitors` cleanly. |
