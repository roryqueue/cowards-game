# v1.22 Requirements: WASM/WASI Multi-Compiler Alpha and Runtime Hardening

## Goal

Make WASM/WASI a more serious multi-compiler Strategy runtime candidate by adding honest Zig alpha proof, strengthening runtime hardening evidence, and documenting ABI evolution tradeoffs without promoting Rust, Zig, or WASM/WASI to counted play or production sandbox certification.

## Baseline

- v1.21 is complete, committed, tagged `v1.21`, and archived.
- JS/TS remains the counted Strategy path.
- Python remains non-counted exhibition beta.
- Rust WASM/WASI remains non-counted exhibition alpha and is the regression floor.
- Zig must fail loud unless binary detection, target support, immutable WASM artifact metadata, Wasmtime execution, and ABI-valid Match behavior all pass.
- Go owns orchestration, persistence-facing backend behavior, Match lifecycle, scoring, public evidence, retry policy, and promotion decisions.
- Runtime-service owns hostile Strategy execution only through schema-validated ABI envelopes and registered runtime implementations.

## Requirements

1. Preserve the Rust WASM/WASI compile and Wasmtime execution path from v1.21.
2. Preserve JS/TS counted Strategy support and Python non-counted exhibition beta support.
3. Fix Zig preflight so shell-visible `zig` is detected through honest PATH-aware tooling rather than an empty environment false negative.
4. Require Zig `wasm32-wasi` compile proof before exposing Zig as working.
5. Compile Zig source to immutable WASM artifact metadata with hash, byte count, target triple, WASI profile, ABI envelope, toolchain evidence, and public non-counted evidence.
6. Execute Zig artifacts only through runtime-service / Runtime Broker / Wasmtime.
7. Keep Zig unavailable/non-promoted in environments where compile, runtime, or ABI proof fails.
8. Add no-std Zig starter samples only after proof passes.
9. Keep Rust/Zig/WASM non-counted and out of ranked, ladder, gauntlet, and counted Match eligibility.
10. Harden Wasmtime evidence for timeout/fuel, memory cap, stdout/stderr/result caps, malformed JSON, invalid schema, trap/panic/abort, hash mismatch, missing artifact behavior, no inherited env, no preopens, and public-safe diagnostics.
11. Strengthen artifact compatibility metadata around target triples, WASI profile, ABI envelope, hashes, stale metadata, rollback, and retention language.
12. Document ABI evolution options for current Preview 1 stdin/stdout JSON, direct exports, and component model/WIT.
13. Keep Preview 1 stdin/stdout JSON as the only v1.22 execution ABI unless a separate proof is explicitly isolated and non-promotional.
14. Produce v1.22 evidence artifacts and promotion decision artifacts.
15. Verify `/workshop`, Workshop validation/save paths, account revision save paths, non-counted labels, public MatchSet/replay evidence copy, and no silent fallback behavior.
16. Close by archiving v1.22 roadmap, requirements, phases, audit, and evidence; remove active `.planning/REQUIREMENTS.md`.

## Non-Goals

- No Rust, Zig, WASM/WASI, or Python counted/ranked/ladder/gauntlet promotion.
- No production sandbox certification claim.
- No Rust, Zig, Python, or TypeScript backend ownership creep.
- No Strategy code execution in web/API/Go.
- No arbitrary Cargo/Zig package install support.
- No direct-export or component-model execution path in v1.22.
