# Plan 12-01: Docker Service Readiness and Shared Preflight

## Objective

Add a shared preflight command that checks service startup, migrations, smoke seed/worker execution, and optional replay endpoint readiness with layer-specific diagnostics.

## Tasks

1. Add `scripts/wait-for-compose-services.sh` to start or inspect Compose services and wait for `healthy` status for Postgres and Redis.
2. Add `scripts/preflight.ts` with labelled checks for:
   - service startup: Postgres connection
   - service startup: Redis availability when required
   - migration: migrate current database
   - seeding/worker execution: run `runDevelopmentMatchSetSmoke`
   - replay projection: verify at least one Chronicle-backed replay can be built from persisted data
   - UI rendering: optionally fetch a configured replay URL when a web server URL is provided
3. Add scripts for `preflight`, `preflight:docker`, and `preflight:local`.

## Verification

- `pnpm preflight -- --skip-redis --skip-web`
- `pnpm preflight:docker -- --skip-web`
- `pnpm format:check`

## Done

Preflight output clearly reports pass/fail by layer and exits nonzero on failed required checks.
