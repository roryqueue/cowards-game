# Plan 09-03: Semantic Event Grammar State Machine

## Objective

Create a pure semantic Chronicle grammar validator for legal Match/Round/Activation/Cycle event windows and event context consistency.

Purpose: This implements GRAM-01, GRAM-02, GRAM-03, and the corrupted-fixture portion of GRAM-08 while honoring D-01, D-02, D-03, D-04, D-10, and D-11. The validator uses Chronicle events and payload semantics only; it does not run Strategy source or become a second engine.

Output: A tested `packages/replay/src/grammar.ts` module that Plan 09-05 can wire into `validateChronicle`.

    Create `packages/replay/src/grammar.ts` with a pure `validateChronicleGrammar(chronicle)` function. Track explicit Match, Round, Activation, and terminal state while iterating events in sequence order. Return validation errors; do not throw.

<task type="auto" tdd="true">
  <name>Task 2: Enforce required event context fields</name>
  <files>packages/replay/src/grammar.ts, packages/replay/src/grammar.test.ts</files>
  <behavior>
    - Round-scoped events require `context.roundNumber` matching the active Round.
    - Activation-scoped events require `context.activationId`, `context.activationIndex`, `context.actingPlayerId`, and `context.soldierId` where the event type acts on a Soldier.

<task type="auto" tdd="true">
