# Plan 13-02: Workshop Owner Replay Links

## Objective

Expose the authorized owner-debug replay path from real Workshop Match results while preserving the existing public replay link.

## Read First

- `.planning/phases/13-close-gap-persisted-owner-replay-debug-authorization/13-UI-SPEC.md`
- `packages/persistence/src/matchset-status.ts`
- `packages/persistence/src/workshop.ts`
- `apps/web/app/workshop/workshop-client-state.ts`
- `apps/web/app/workshop/workshop-client.tsx`

## Tasks

1. Extend MatchSet match summaries with bottom/top participant ids or an equivalent server-derived field needed to identify whether the local Workshop player owns the Match.
2. Add Workshop helper functions that build:
   - the existing public replay href;
   - an owner-debug replay href with `ownerDebug=1` and the actual local Workshop participant id.
3. Render `Open owner debug` beside `Open replay` only when a replay exists and the local Workshop player is a participant.
4. Add helper/UI tests, including a mirrored Match where `player:workshop-local` is top.

## Threat Model

- UI links request owner debug but do not authorize it.
- No private owner DTOs or Strategy/runtime internals are displayed in Workshop Match rows.

## Verification

- `pnpm --filter @cowards/persistence test -- workshop.test.ts`
- `pnpm --filter @cowards/web test -- workshop-client.test.tsx`

## Done

Workshop results provide both public and owner-debug replay paths for completed persisted Matches, using actual participant identity rather than side assumptions.
