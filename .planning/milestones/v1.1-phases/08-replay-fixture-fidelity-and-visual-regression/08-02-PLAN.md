# Plan 08-02: Fixture Legality and Layer Diagnostics

## Objective

Add package-level legality and validation checks that reject impossible canonical replay beats before `apps/web` can render them.

Purpose: This implements FID-03 and the engine/Chronicle portions of FID-07 while keeping game-rule checks outside React per AGENTS.md.

Output: Focused tests that classify failures as `[engine legality]` or `[Chronicle validation]`.

<task type="auto" tdd="true">
  <name>Task 2: Assert mechanic legality from reconstructed states</name>
  <files>packages/test-utils/src/replay-scenarios.legality.test.ts</files>
  <behavior>
    - `[engine legality] push scenario` proves push resolves into an empty legal destination or produces legal fallout.
    - `[engine legality] fall scenario` proves a Soldier transitions from ACTIVE or STONE to FALLEN with null position.
    - `[engine legality] contraction scenario` proves bounds shrink and every non-FALLEN Soldier outside new bounds becomes FALLEN.
    - `[engine legality] legal-backstab scenario` proves every `BACKSTAB_RESOLVED` pair has an active attacker directly behind an active enemy victim in the previous reconstructed board.
    - `[engine legality] runtime-failure scenario` proves owner-private `RUNTIME_VIOLATION` appears without public Strategy source or memory payloads.
    - `[engine legality] endgame scenario` proves one terminal `MATCH_ENDED` and the final replay state has an outcome.
  </behavior>
  <action>
    Add helper functions inside the test file for board lookup, position comparison, behind-square calculation, and payload record parsing. Keep these helpers test-only; do not move game rules into `apps/web`.

    Use reconstructed replay states from `createReplay` rather than raw snapshots whenever possible, so failures prove replay application as well as scenario generation. Prefix test names with `[engine legality]` per D-08 and D-09.
  </action>
  <verify>
    <automated>pnpm --filter @cowards/test-utils test -- replay-scenarios.legality.test.ts -t "[engine legality]"</automated>
  </verify>
  <done>Each canonical mechanic is proven legal from engine-generated Chronicle data and reconstructed replay states.</done>
</task>
