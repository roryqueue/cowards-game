# Plan 08-03: Replay Fixture Projection Parity

## Objective

Refactor the web replay fixture path so canonical generated fixture Chronicles are converted to `ReplayReadyDto` through the same projection and reconstruction path used by persisted Match Chronicles.

Purpose: This implements FID-04 and the projection/DTO portion of FID-07 while preserving public replay privacy.

Output: Shared Chronicle-to-ready-replay helper, generated fixture lookup by scenario ID, and projection parity tests.
