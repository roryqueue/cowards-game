# Plan 08-04: Focused Replay Visual Regression

## Objective

Add deterministic Playwright visual checks for generated replay fixtures at stable desktop and mobile viewports.

Purpose: This implements FID-05 and the UI-rendering portion of FID-07 using focused mechanic screenshots per D-05, D-06, and D-07.

Output: A runnable visual regression script and focused screenshots for board scale, Soldier positions, contraction bounds, and event callouts.

  <files>apps/web/e2e/replay.visual.spec.ts, apps/web/e2e/replay.visual.spec.ts-snapshots/</files>
  <behavior>
    - `[ui rendering] board scale` captures the replay board locator for the generated `compound-tour` or `push` scenario at desktop and mobile viewport projects.
    - `[ui rendering] Soldier positions` selects a checkpoint from `push` and screenshots only the board locator after the target event is selected.
    - `[ui rendering] contraction bounds` selects a checkpoint from `contraction` and screenshots only the board locator.
    - `[ui rendering] event callouts` captures focused board screenshots for `push`, `legal-backstab`, `runtime-failure`, and `endgame` callouts.
    - Every screenshot first waits for one visible canvas and verifies nonblank canvas pixels.
  </behavior>
  <action>
    Create `apps/web/e2e/replay.visual.spec.ts` using Playwright `expect(locator).toHaveScreenshot(...)` against focused locators, not full-page screenshots.

    Use the fixture route scenario catalog from Plan 08-03. Navigate directly to each scenario's `replayHref`, click stable `Timeline event {sequence}: {eventType}` buttons, and assert the board host label includes the selected sequence/event before screenshot capture.

    Disable animations in screenshot options and set a small nonzero threshold such as `maxDiffPixelRatio: 0.01`. Keep screenshot names deterministic and mechanic-specific, for example `replay-board-push-callout.png`, `replay-board-contraction-bounds.png`, and include project-specific baselines generated by Playwright under `apps/web/e2e/replay.visual.spec.ts-snapshots/`.

    If Pixi animation still causes variance, set the timeline input while `scrubbing` is active or wait for the board status text to settle before screenshotting; do not broaden screenshots to the full page.
  </action>
