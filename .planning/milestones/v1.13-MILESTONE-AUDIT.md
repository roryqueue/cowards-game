# v1.13 Milestone Audit

**Milestone:** Go Backend Ownership Cutover
**Date:** 2026-05-23
**Result:** Pass with accepted fork-route deferral.

## Findings

No open audit findings remain.

## Accepted Deferrals

- Starter and Advanced fork ownership did not promote. This is accepted because parity-safe Go fork implementation needs library source manifest access that is not present in the Go backend yet. The selected web path keeps forks TypeScript-owned by default, and Go fork handlers fail closed rather than silently falling back.
- TypeScript worker/runtime ownership remains by design for job claiming, Strategy execution, Match completion, Chronicle generation, scoring completion, and runtime failure classification.
- Full replay projection, owner-debug replay, Go migrations, production sandbox promotion, and counted non-JS play remain outside v1.13.

## Issues Found And Fixed

- Cross-account Strategy Revision writes initially trusted a client `strategyId`; fixed with transaction-scoped ownership proof.
- Exhibition creation initially missed TypeScript parity for rate limits, counted runtime eligibility, product preset expansion, and arena seeding; fixed in Go.
- Go-created MatchSet summaries initially normalized in-progress statuses too aggressively; fixed to preserve route-safe status evidence and replay availability.
- Production auth cookies initially lacked secure-cookie enforcement; fixed for production or explicit secure-cookie mode.
- Worker preflight could claim unrelated queued work in a reused dev database; fixed by allowing a Match ID claim filter.
- Public player live validation found schema-invalid exhibition result `seasonId: ""`; fixed by returning only ladder-linked player result summaries from Go.
