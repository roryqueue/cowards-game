# Plan 07-04: Workshop Replay Handoff

<objective>
Extend the Workshop test panel so completed or degraded MatchSets show a full Match list with status, outcome, aggregate scoring, replay availability, and direct links to `/matches/{matchId}/replay` only when a stored Chronicle exists.
</objective>

<requirements_addressed>
VIEW-07
</requirements_addressed>

<threat_model>
| Threat | Severity | Mitigation |
|--------|----------|------------|
| T-07-07: Workshop renders broken replay links before Chronicles exist | medium | Match summaries include `hasReplay`; links render only for completed Matches with stored Chronicles. |
| T-07-08: Degraded MatchSets hide completed replay evidence | medium | Degraded summaries keep completed Match rows and mark failed/system Matches separately. |
| T-07-09: MatchSet scoring is duplicated differently in UI | medium | UI consumes existing `getWorkshopTestSummary` scoring DTO and only formats it. |
</threat_model>

<tasks>
<task id="07-04-01" type="execute">
  <title>Enrich MatchSet summaries with outcomes and replay availability</title>
  <read_first>
    - packages/persistence/src/matchset-status.ts
    - packages/persistence/src/workshop.ts
    - packages/persistence/src/chronicle-store.ts

    - .planning/phases/07-replay-viewer-and-end-to-end-verification/07-CONTEXT.md
  </read_first>
  <action>
    Update persistence summary types so each Workshop Match row includes:
    - `matchId`
    - `status`
    - `outcome` when available
    - `winnerPlayerId` when available
    - `hasReplay: boolean`

    In `packages/persistence/src/workshop.ts`, update `getWorkshopTestSummary(pool, matchSetId)` and the launch response from `createWorkshopTestMatchSet(pool, input)` so `matches` contains the enriched row shape.

    Determine `hasReplay` by checking stored Chronicle rows for each Match ID, using the existing Chronicle table/store patterns. Do not infer replay availability from Match status alone.

    Preserve degraded MatchSet behavior: completed Matches keep outcome and `hasReplay`, failed/system Matches show failure status with `hasReplay: false`.

    Add or update `packages/persistence/src/workshop.test.ts` proving:
    - summary rows include `matchId`, `status`, and `hasReplay`
    - completed rows with stored Chronicles have `hasReplay: true`
    - completed rows without stored Chronicles have `hasReplay: false`
    - failed/system rows do not get replay availability
  </action>
  <verify>

    pnpm --filter @cowards/persistence typecheck
    rg 'hasReplay' packages/persistence/src/workshop.ts packages/persistence/src/workshop.test.ts
    rg 'winnerPlayerId|outcome' packages/persistence/src/workshop.ts
  </verify>
  <acceptance_criteria>
    - `packages/persistence/src/workshop.ts` contains `hasReplay`
    - `packages/persistence/src/workshop.ts` contains `outcome`

    - `packages/persistence/src/workshop.test.ts` asserts `hasReplay: false`
    - `pnpm --filter @cowards/persistence test -- workshop.test.ts` exits 0
  </acceptance_criteria>
</task>

<task id="07-04-02" type="execute">
  <title>Expose replay links in Workshop MatchSet test panel</title>
  <read_first>

  </read_first>
  <action>
    Update `apps/web/app/workshop/types.ts` so `WorkshopTestSummary.matches` includes `outcome`, `winnerPlayerId`, and `hasReplay`.

    Add helper functions in `workshop-client-state.ts`:
    - `getReplayHref(matchId: string): string` returning `/matches/${encodeURIComponent(matchId)}/replay`
    - `canOpenReplay(match)` returning true only when `match.status === "complete"` and `match.hasReplay === true`
    - `formatMatchOutcome(match)` returning a compact outcome label from `outcome`/`winnerPlayerId` or `Pending`, `Running`, `Failed system`, `Replay unavailable`

    Update `workshop-client.tsx` test panel to render the full Match list for completed and degraded MatchSets:
    - each row shows status and compact outcome
    - completed rows with `hasReplay` render a link with text `Open replay`
    - failed/system rows render failure status and no link
    - degraded MatchSets show text `Some Matches failed, but completed replays are available.`
    - aggregate scoring remains visible

    Keep this in the existing Workshop panel; do not create a separate match-history page.
  </action>
  <verify>
    pnpm --filter @cowards/web test -- workshop-client.test.tsx
    pnpm --filter @cowards/web typecheck

  </verify>
  <acceptance_criteria>

    - `workshop-client.tsx` contains `Open replay`
    - `workshop-client.tsx` contains `Some Matches failed, but completed replays are available.`

    - `pnpm --filter @cowards/web test -- workshop-client.test.tsx` exits 0
