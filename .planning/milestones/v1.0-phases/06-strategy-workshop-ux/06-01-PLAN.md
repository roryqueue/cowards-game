# Plan 06-01: Workshop Data and API Foundation

<objective>
Create the server-side Workshop foundation: dependencies, local seed helpers, UI-safe DTOs, validation/submission/history/test APIs, and tests proving the web/API process validates and persists Strategy source without executing Strategy runtime behavior.
</objective>

<requirements_addressed>
UX-01, UX-02, UX-03, UX-04, UX-05, UX-06
</requirements_addressed>

<threat_model>
| Threat | Severity | Mitigation |
|--------|----------|------------|
| T-06-01: Web/API process executes hostile Strategy code while validating or launching tests | high | Only call `validateStrategySource` and `buildStrategyRevision` in web/server code; never import `@cowards/runtime-js/worker`, `createRuntimeFromRevision`, or engine execution APIs from `apps/web`. |
| T-06-02: Workshop UI invents throwaway data and cannot feed Phase 7 replay | high | Route revision submission and test launch through persisted StrategyRevision and MatchSet services, returning real revision IDs, MatchSet IDs, statuses, and scoring. |
| T-06-03: Label/notes metadata silently violates StrategyRevision schema | medium | Either extend `StrategyRevisionMetadata` with optional `notes` in spec/runtime tests, or omit notes from first UI. This plan chooses to add `notes?: string`. |
</threat_model>

    - `WorkshopInitialData`
    - `WorkshopTemplateDto`
    - `WorkshopRevisionDto`
    - `WorkshopValidationResponse`
    - `WorkshopSubmitRevisionRequest`
    - `WorkshopSubmitRevisionResponse`
    - `WorkshopLaunchTestRequest`
    - `WorkshopTestDto`

    Add `apps/web/app/workshop/server.ts` exporting server-only functions:
    - `getWorkshopInitialData(): Promise<WorkshopInitialData>`
    - `validateWorkshopSource(source: string): WorkshopValidationResponse`
    - `submitWorkshopRevision(input: WorkshopSubmitRevisionRequest): Promise<WorkshopSubmitRevisionResponse>`
    - `launchWorkshopTest(input: WorkshopLaunchTestRequest): Promise<WorkshopTestDto>`
    - `getWorkshopTest(matchSetId: string): Promise<WorkshopTestDto>`

    Server function rules:
    - Create and close a database pool per request path unless a test dependency override is injected.
    - Call `validateStrategySource(source)` for validation.
    - Build submissions with `buildStrategyRevision({ source, strategyId: WORKSHOP_STRATEGY_ID, metadata: { createdBy: WORKSHOP_USER_ID, label, notes } })`.
    - Reject submission when `revision.validation.valid !== true` with an error response containing `Resolve validation errors before submitting.`
    - Call `ensureWorkshopSeed` before reads/writes that need local user/strategy/opponent/arena data.
    - Do not import `@cowards/runtime-js/worker`, `createRuntimeFromRevision`, `@cowards/engine`, or `@cowards/replay` anywhere under `apps/web`.

    Add route handlers:
    - `apps/web/app/api/workshop/route.ts` with `GET` returning `getWorkshopInitialData()`.
    - `apps/web/app/api/workshop/validate/route.ts` with `POST` accepting `{ source: string }` and returning `validateWorkshopSource(source)`.
    - `apps/web/app/api/workshop/revisions/route.ts` with `POST` accepting `WorkshopSubmitRevisionRequest` and returning `submitWorkshopRevision(...)`.
    - `apps/web/app/api/workshop/tests/route.ts` with `POST` accepting `WorkshopLaunchTestRequest` and returning `launchWorkshopTest(...)`.
    - `apps/web/app/api/workshop/tests/[matchSetId]/route.ts` with `GET` returning `getWorkshopTest(matchSetId)`.

    Route handlers must return `Response.json(...)` and use HTTP 400 for invalid request shapes or invalid submissions.
