# Plan 03-03: Replay Reconstruction, Validation, and Integrity

<objective>
Implement replay utilities that normalize, hash, validate, and reconstruct Chronicle board truth through `stateAt(sequence)` and a linear iterator without re-executing strategy source.
</objective>

<requirements_addressed>
REPLAY-03, REPLAY-04, REPLAY-05, TEST-03
</requirements_addressed>

<threat_model>
| Threat | Severity | Mitigation |
|--------|----------|------------|
| T-03-05: Replay silently accepts corrupted Chronicles | high | Validate schema, versions, required events, broad ordering, snapshots, and hash integrity with typed errors. |
| T-03-06: Reconstruction reruns strategies or engine transitions as hidden source of truth | high | Reconstruct from Chronicle snapshots/events only and add tests using a Chronicle without a runtime object. |
| T-03-07: Hash changes because storage metadata or insertion order changes | medium | Normalize deterministic content and exclude `storageMetadata` from hash input. |
</threat_model>

<task id="03-03-02" type="execute">
  <title>Implement typed Chronicle validation</title>
  <read_first>
    - packages/replay/src/validate.ts
    - packages/replay/src/hash.ts
    - packages/spec/src/types.ts
    - packages/spec/src/schemas.ts

  </read_first>
  <action>
    Add `packages/replay/src/validate.ts` exporting `validateChronicle(chronicle)` and `assertChronicleCompatible(chronicle)`.

    Validation must return `ChronicleValidationResult` and produce typed errors for:
    - `SCHEMA_INVALID` when `ChronicleSchema.safeParse` fails
    - `VERSION_INCOMPATIBLE` when `schemaVersion !== "chronicle-v1"` or chronicle/spec versions are unsupported
    - `EVENT_ORDER_INVALID` when sequence values are not contiguous from zero or broad order is violated
    - `REQUIRED_EVENT_MISSING` when any required Phase 3 event type is absent from a completed match
    - `SNAPSHOT_MISSING` when boundary snapshots required by D-02 are absent
    - `HASH_MISMATCH` when provided integrity does not match `createChronicleContentHash`
    - `UNSUPPORTED_MIGRATION` through an exported `migrateChronicle()` hook that currently supports no real migrations
  </action>

<task id="03-03-03" type="execute">
  <title>Implement stateAt and linear replay iterator</title>
  <read_first>
    - packages/replay/src/reconstruct.ts
    - packages/replay/src/validate.ts
    - packages/replay/src/build.ts
    - packages/spec/src/types.ts
  </read_first>
  <action>
    Add `packages/replay/src/reconstruct.ts` exporting `createReplay(chronicle)`, `stateAt(sequence)`, and `iterateReplay()`.

    `createReplay(chronicle)` must validate the Chronicle first. If validation fails, return `{ ok: false, errors }`. If validation passes, return `{ ok: true, replay }`.

    `stateAt(sequence)` must find the nearest boundary snapshot at or before the target sequence, apply supported public semantic event effects until the target sequence, and return `{ ok: true, state }` or `{ ok: false, errors }`. Supported Phase 3 event effects must include movement advance/block, turn, push resolved/blocked, Backstab resolved, Soldier stoned, Soldier fell, contraction, and match end. Unsupported event effects that do not change board truth, such as `AWARENESS_GRID_OBSERVED`, must be skipped explicitly.

    `iterateReplay()` must yield ordered `{ sequence, event, state }` entries for every event sequence.
  </action>
  <verify>
    pnpm --filter @cowards/replay test -- reconstruct.test.ts
    rg "export const createReplay" packages/replay/src/reconstruct.ts
    rg "stateAt" packages/replay/src/reconstruct.ts
    rg "iterateReplay" packages/replay/src/reconstruct.ts
  </verify>
  <acceptance_criteria>
    - `packages/replay/src/reconstruct.ts` contains `export const createReplay`
    - `packages/replay/src/reconstruct.ts` contains `stateAt`
    - `packages/replay/src/reconstruct.ts` contains `iterateReplay`
    - `packages/replay/src/reconstruct.ts` does not contain `runSoldierBrain`
    - `pnpm --filter @cowards/replay test -- reconstruct.test.ts` exits 0
  </acceptance_criteria>
</task>
