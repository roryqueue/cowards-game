# Plan 03-04: Public and Owner Replay Projections

<objective>
Implement projection utilities that derive privacy-safe public Chronicles and per-player owner debug views from one canonical Chronicle artifact.
</objective>

<requirements_addressed>
REPLAY-06, REPLAY-07, TEST-03
</requirements_addressed>

<threat_model>
| Threat | Severity | Mitigation |
|--------|----------|------------|
| T-03-06: Public projection exposes exact grids, objectives, memory, source, or raw runtime details | high | Stringify public projections in tests and assert known private marker values are absent. |
| T-03-07: Owner projection exposes opponent private debug data | high | Owner projection requires `playerId` and returns only `private.byPlayerId[playerId]`. |
| T-03-08: Projection mutates canonical Chronicle | medium | Implement pure projection functions and freeze/clone test fixtures before projection. |
</threat_model>

<tasks>
<task id="03-04-01" type="execute">
  <title>Implement public and owner projection utilities</title>
  <read_first>
    - packages/replay/src/project.ts
    - packages/replay/src/build.ts
    - packages/spec/src/types.ts
    - .planning/phases/03-chronicle-and-replay-core/03-CONTEXT.md
  </read_first>
  <action>
    Add `packages/replay/src/project.ts` exporting:
    - `projectChronicle(chronicle, viewer)`
    - `projectPublicChronicle(chronicle)`
    - `projectOwnerChronicle(chronicle, playerId)`

    `projectPublicChronicle` must return schema-valid board truth and public events while removing `private`, `privateRef`, exact Awareness Grids, objective payloads, StrategyMemory, SoldierMemory, strategy source, and raw runtime details. Public runtime violations may expose only `type`, `category`, affected sequence/context, and owning soldier/player IDs when present.

    `projectOwnerChronicle` must return public content plus only `chronicle.private.byPlayerId[playerId]`; it must not include the opposing player's private entries.
  </action>
  <verify>
    pnpm --filter @cowards/replay test -- project.test.ts
    rg "export const projectChronicle" packages/replay/src/project.ts
    rg "projectPublicChronicle" packages/replay/src/project.ts
    rg "projectOwnerChronicle" packages/replay/src/project.ts
  </verify>
  <acceptance_criteria>
    - `packages/replay/src/project.ts` contains `export const projectChronicle`
    - `packages/replay/src/project.ts` contains `export const projectPublicChronicle`
    - `packages/replay/src/project.ts` contains `export const projectOwnerChronicle`
    - `packages/replay/src/index.ts` exports from `./project.js`
    - `pnpm --filter @cowards/replay test -- project.test.ts` exits 0
  </acceptance_criteria>
</task>
