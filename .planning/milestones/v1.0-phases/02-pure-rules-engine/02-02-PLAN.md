# Plan 02-02: Round Loop, Runtime Boundary, and Activation Selection

<objective>
Implement the synchronous fake runtime boundary, runtime input builders, activation order filtering, no-advance cleanup, and deterministic Round 1-4 activation loop.
</objective>

<requirements_addressed>
ENG-02, ENG-03, ENG-04, ENG-05, ENG-06, ENG-16, ENG-21, TEST-01
</requirements_addressed>

<threat_model>
| Threat | Severity | Mitigation |
|--------|----------|------------|
| T-02-03: Runtime failures crash or escape the deterministic engine path | high | Runtime methods return `RuntimeResult<T>`; violations become transition summaries and activation interruptions. |
| T-02-04: Activation ordering differs between runs | high | Implement deterministic seed-derived initial initiative, alternating initiative per round, and explicit snake ordering tests. |
| T-02-05: Invalid activation selections execute illegal Soldiers | medium | Filter duplicates, malformed orders, STONE/FALLEN Soldiers, and excess orders before activation. |
</threat_model>
