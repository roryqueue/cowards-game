# Plan 02-04: Contraction, Match End, and Invariant Matrix

<objective>
Implement contraction, immediate match-ending semantics, final 2x2 resolution, and deterministic invariant matrix coverage for board/state validity.
</objective>

<requirements_addressed>
ENG-17, ENG-18, ENG-19, ENG-20, ENG-21, TEST-02
</requirements_addressed>

<threat_model>
| Threat | Severity | Mitigation |
|--------|----------|------------|
| T-02-06: Contraction leaves occupied squares outside bounds | high | Dedicated contraction transition updates bounds, falls out-of-bounds ACTIVE/STONE Soldiers, removes out-of-bounds TerrainStones, and runs invariant tests. |
| T-02-07: Match end conditions are checked too late | high | Check immediate win/draw after every activation and after contraction/final 2x2. |
| T-02-08: State invariants regress silently | medium | Add deterministic fixture matrix for occupancy uniqueness, bounds validity, status semantics, and ordering. |
</threat_model>

    - `getContractedBounds(bounds): BoardBounds`
    - `resolveContraction(state): TransitionResult`
    - `checkFinalTwoByTwoResolution(state): MatchOutcome | undefined`

    `resolveContraction` must:
    - run normal win condition before contraction
    - shrink bounds by incrementing `minX`/`minY` and decrementing `maxX`/`maxY`
    - make out-of-bounds ACTIVE or STONE Soldiers FALLEN with reason `BOARD_CONTRACTION`

    - check match end after contraction
    - when board reaches 2x2, count ACTIVE Soldiers inside bounds and produce WIN or DRAW
    - emit `CONTRACTION_RESOLVED`, `SOLDIER_FELL`, and `MATCH_ENDED` summaries as applicable
  </action>
