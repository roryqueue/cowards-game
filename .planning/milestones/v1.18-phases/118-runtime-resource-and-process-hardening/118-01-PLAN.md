# Phase 118 Plan: Runtime Resource and Process Hardening

## Goal

Harden Python subprocess launch and deterministic failure classification while preserving JS/TS support and no-fallback behavior.

## Tasks

- Centralize Python host executable, isolated args, and empty environment.
- Launch Python host with `-I`, empty env, and `shell: false`.
- Map timeout, stdio cap, subprocess signal/exit, and malformed IPC deterministically.
- Add tests for launch args, timeout, and output flood classification.
- Extend monitors to fail if v1.18 hardening markers drift.
