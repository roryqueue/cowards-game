# Phase 170 Plan

## Goal

Spike Component Model/WIT as a future ABI option without changing Match execution.

## Scope

In scope:
- Component Model/WIT proof or fail-loud non-promotion artifact.
- Minimal Strategy WIT world/interface shape and generated binding/tooling status.
- Wasmtime host integration, import surface, timeout/fuel, memory, result cap, validation, trap handling, and privacy behavior.
- Rust and Zig parity attempt where feasible.
- Compatibility and rollback impact against Preview 1 JSON artifact metadata.

Out of scope:
- Making Component Model/WIT artifacts eligible for Match execution.
- Promoting WASI 0.2 or WIT over the active Preview 1 JSON ABI.

## Tasks

1. Define the smallest Strategy WIT world/interface needed to represent the current ABI envelope.
2. Probe Rust Component Model tooling first and Zig tooling where feasible.
3. Record Wasmtime host integration approach and resource-cap story.
4. Capture validation, trap, import-surface, privacy, and no-fallback behavior.
5. Generate WIT proof and non-promotion artifacts.
6. Add guard evidence that WIT artifacts are not Match execution artifacts.
