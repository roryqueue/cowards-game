# Phase 168 Plan

## Goal

Produce a public-safe matrix of what each runtime lane proves, does not prove, and needs before stronger sandbox claims.

## Scope

In scope:
- Matrix rows for JS/TS, Python, Rust, Zig, WASM/WASI, direct exports, and Component Model/WIT.
- Evidence tiers for local, CI, signed-in, operational/deployment, and external-review proof.
- No-fallback status for stopped, unavailable, stale, mismatched, malformed, and capability-invalid lanes.
- Public-safe summaries excluding private diagnostics and runtime internals.

Out of scope:
- Granting production sandbox certification by implication.
- Changing runtime promotion status.

## Tasks

1. Gather baseline, taxonomy, JS/Python, and WASM/WASI evidence artifacts.
2. Define matrix fields for proves, does-not-prove, gaps, no-fallback status, privacy status, and promotion status.
3. Generate JSON and Markdown readiness matrix artifacts.
4. Record direct exports and Component Model/WIT as spike-only/not-active unless later evidence explicitly changes that.
5. Add a check to catch public summary leaks and overclaiming language.
