# Phase 171 Plan

## Goal

Decide active ABI status and write fail-closed migration and rollback criteria.

## Scope

In scope:
- Explicit ABI decision keeping or changing active WASM/WASI execution ABI.
- Rationale for not promoting, or explicitly promoting, direct exports and Component Model/WIT.
- Fail-closed criteria for unknown, stale, mismatched, or unpromoted ABI metadata.
- Migration criteria for compatibility, schema, caps, privacy, replay, no-fallback, rollback, and artifact coexistence.
- Ownership statement preserving Go orchestration and runtime-service hostile execution boundaries.

Out of scope:
- Silent ABI migration.
- Strategy execution in web/API/Go.
- Counted non-JS promotion.

## Tasks

1. Read direct-export and Component Model/WIT proof artifacts.
2. Compare spike evidence against Preview 1 stdin/stdout JSON compatibility and rollback needs.
3. Write the v1.24 ABI decision artifact.
4. Define fail-closed metadata gates for future ABI experimentation.
5. Verify monitors and evidence commands still treat unpromoted ABI metadata as ineligible.
