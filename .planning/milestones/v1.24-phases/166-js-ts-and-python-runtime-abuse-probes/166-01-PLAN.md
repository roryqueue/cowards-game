# Phase 166 Plan

## Goal

Run and record abuse/no-fallback probes across JS/TS counted and Python non-counted exhibition beta lanes.

## Scope

In scope:
- JS/TS probes for timeout, oversized output, malformed result, invalid schema/action, thrown error, runtime unavailability, and forbidden capabilities where applicable.
- Python probes for timeout, oversized output, malformed result, invalid schema/action, crash, runtime unavailability, forbidden filesystem/network/process/import/env/time/random/host-path behavior where applicable.
- No-fallback evidence for stopped, unavailable, malformed, and capability-invalid lanes.
- Public-safe evidence preserving JS/TS counted and Python non-counted semantics.

Out of scope:
- WASM/WASI Rust/Zig probes.
- Changing eligibility or promotion status.

## Tasks

1. Extend the v1.24 abuse schema into JS/TS and Python probe definitions.
2. Add or update local evidence commands for JS/TS and Python abuse probes.
3. Ensure runtime unavailability and invalid probe outcomes do not execute Strategy code in web/API/Go.
4. Record Python non-counted exhibition beta no-fallback behavior.
5. Generate JSON and Markdown evidence artifacts.
6. Add focused tests for failure classification and public-safe output.
