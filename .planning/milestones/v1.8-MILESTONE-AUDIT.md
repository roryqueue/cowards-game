# v1.8 Milestone Audit — Production Boundary Hardening

**Audited:** 2026-05-22
**Status:** Passed

## Scope

v1.8 hardened the v1.7 service, runtime, and backend contracts without moving orchestration ownership, rewriting the backend, or promoting production multi-language Strategy execution.

## Integration Notes

- `@cowards/spec` is the canonical source for service routes, public DTO schemas, runtime product semantics, and counted eligibility.
- `@cowards/service` remains the TypeScript-owned service boundary for local behavior.
- Go serves only the v1.8 read-only allowlist from generated parity fixtures.
- Strategy execution remains outside the web/API process, and sandbox work remains evaluation-only.
- Python and other non-JS runtimes remain experimental and non-counted.
- Broad web persistence/runtime debt remains visible and is baseline-gated so new bypass imports fail.

## Surprises

- Public fixture prose intentionally names omitted private categories such as StrategyMemory; privacy checks now distinguish explanatory text from forbidden DTO fields.
- The topology harness needed URL-secret redaction even for local diagnostics, because local URLs can carry tokens in real developer workflows.
- The broad web import scan was too noisy to flip strict all at once, but statement-fingerprint baselining makes it useful immediately.
- Runtime adapter drift needed product semantics checks, not just adapter metadata checks, to avoid false confidence.

## Deferred By Design

- Go mutation endpoints, job claiming, persistence writes, migrations, Match orchestration, and Strategy execution.
- Production hostile-code sandbox certification.
- Counted non-JS MatchSets, ladders, or gauntlets.
- Public language picker and package dependency ecosystem.
- Cloud deployment, service mesh, Kubernetes, or production observability stack.

## Recommendation For v1.9

Shape v1.9 around one ownership move instead of three at once. The best next milestone is **Backend and Runtime Ownership Split**, with phases:

1. Pick the first production boundary to move: Go read model expansion, runtime isolation service, or mutation boundary design.
2. Replace report-only web persistence debt with service-backed slices until strict import enforcement can widen.
3. Promote a sandbox candidate only after resource, filesystem, network, deployment, and failure-taxonomy evidence is stronger.
4. Keep Go writes out of scope unless route ownership, auth, jobs, migrations, and rollback semantics are specified first.
5. Define promotion criteria for non-JS counted play, but keep Python experimental until those criteria are met.
