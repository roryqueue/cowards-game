## Residual Notes

- The Python runtime is intentionally experimental and not enabled for counted MatchSets.
- The Go backend is intentionally static/read-only for the spike; it proves DTO shape and deployment surface, not orchestration ownership.
- Normal counted play still allows the existing JS/TS adapters; production hostile-code isolation remains deferred beyond v1.7.
