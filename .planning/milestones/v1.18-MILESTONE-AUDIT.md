## Audit Findings And Fixes

| Finding | Severity | Fix |
| --- | --- | --- |
| Python account revision saves lost `sourceFormat`, causing Go to persist Python source as TypeScript metadata. | High | Passed `sourceFormat` through Workshop/account save client, account write boundary, Go client, and Go revision creation. |
| Next page evaluation failed on Python validation host path handling. | High | Replaced Node-only URL file-path conversion with side-effect-light URL pathname decoding for the validation host path. |
| Go exhibition creation used an invalid public counted reason for non-counted exhibitions. | High | Mapped non-counted exhibition status to the existing `non_counted` counted reason and public explanation. |
| MatchSet result pages did not expose enough public counted status evidence for the signed-in proof. | Medium | Added safe public counted metadata to MatchSet result DTOs and page rendering. |
| Python proof initially produced runtime timeouts under the default Go runtime-service limit. | Medium | Raised the default runtime-service timeout from 250 ms to 1000 ms while preserving deterministic timeout classification and cap behavior. |
| Strict live Go fixture topology does not apply to live DB proof data. | Low | Kept aggregate boundary topology and live web/runtime-service topology as gates, documented fixture-mode route 404s as incompatible with the signed-in live DB proof environment. |

## Residual Deferred Items

- Production hostile-code sandbox certification.
- Python ranked/ladder/counted eligibility.
- Arbitrary PyPI/package installs or declared dependency resolution.
- Broad multi-language product support beyond Python exhibition beta.
- WASM/WASI/component-model promotion.
- Cloud deployment, Kubernetes, service mesh, or production observability platform work.

## Conclusion

v1.18 satisfies the milestone goal: the runtime isolation boundary is materially stronger and monitored, JS/TS support remains intact, Python can participate in signed-in non-counted exhibition beta MatchSets through the same broker/runtime ABI, replay evidence is public-safe, and no audit findings remain open for the v1.18 scope.
